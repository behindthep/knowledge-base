ЦФТ - это клиент-серверное приложение, которое обменивается по HTTP протоколу с сервером приложений.

Типы базовых понятий (ТБП) — это ключевая сущность платформы, в которой отражены понятия предметной области или абстрактные классы, необходимые для разработчика.

Полное наименование ТБП для пользователя системы: Кредитный договор

Краткое наименование для обращения к ТБП из кода программы: CREDIT_CONTRACT

Таблица, в которой хранятся все экземпляры ТБП: Z#CREDIT_CONTRACT

ТБП, для которых не создаются таблицы (абстрактные классы):

- Дата / дата и время
- Логика
- Строки различной длины
- Числа различной точности

Реквизиты — это аналоги атрибутов классов в ООП или колонок таблиц в БД.

Основной набор свойств реквизита:

- Полное наименование
- Тип
- Короткое наименование

Для каждого ТБП создаётся интерфейсный пакет типа, который содержит в себе набор низкоуровневых базовых функций и процедур работы с экземплярами ТБП:

- Создание экземпляров
- Удаление экземпляров
- Чтение значений реквизитов
- Изменение значений реквизитов

Платформа развития позволяет автоматически сгенерировать для ТБП:

- Конструктор
- Деструктор
- Операцию редактирования

Представлений для ТБП может быть несколько, в каждом из них будет отображаться разный набор данных. Например, мы можем увидеть полный список кредитных договоров или список договоров, действующих на текущую дату. При этом в системе доступа Платформы мы можем ограничить список доступных пользователю представлений.

Для создания представлений разработчику необходимо знать:

- список отображаемых реквизитов. Например, номер, дату создания и период действия кредитного договора.
- условия выборки для фильтрации экземпляров. Например, все или только действующие на данные момент договоры.
- ограничение доступа по реквизиту. Например, договоры только того филиала банка, в котором работает пользователь.

ТБП может иметь несколько представлений, одно из них можно сделать представлением по умолчанию. Оно используется для перехода в операциях и других представлениях, в случаях когда представление для перехода не указано.

У представления может быть входной фильтр, который позволяет пользователю сразу указать критерии поиска данных. Например, ФИО физ. лица, заключившего договор.

В представлении настраивается сортировка экземпляров по умолчанию, кроме этого в Навигаторе можно выполнить сортировку нужную для решаемой задачи.

В колонках, ссылающихся на другие ТБП автоматически настраиваются переходы в соответствующие ТБП, т.е. по клику в ячейке Навигатор открывает представление в другом ТБП. Представление, в которое будет осуществлен переход может быть задано, в противном случае будет открываться представление по умолчанию.

Платформа дает инструмент изменения цвета шрифта или цвета фона ячеек представления, т.е. когда нам нужно обратить внимание пользователя на значение в ячейке или на всю строку, может быть использован клиент-скрипт.

В представлении может быть ограничен список операций, доступных для выполнения.

Представление может быть унаследовано дочерними типами и в нем возможно включить отображение экземпляров дочерних типов.

Операция — пакет программного кода, разбитый на блоки с набором параметров

Основной список операций в ЦФТ-Платформа развития:

- Конструктор — создаёт новый экземпляр ТБП
- Деструктор — удаляет текущий экземпляр ТБП
- Простая операция — работает над экземпляром, на котором была запущена, используется для редактирования реквизитов экземпляра
- Списочная операция — выполняется над массивом экземпляров. Код операции получает список экземпляров, выделенных в представлении, либо переданных при программном вызове
- Групповая операция — не получает экземпляров, выполнение реализуется с учётом параметров операции
- Операция выбора — используется для вызова другой операции в зависимости от реквизитов экземпляра
- Библиотека — не содержит экранной формы и служит для описания публичных функций, процедур и констант, доступных для программного вызова из других операций

Структура операции:

- Локальные описания (код пишется на PL/Plus, выполняется на сервере БД или на сервере приложений) — содержит переменные, процедуры и функции, которые доступны из тела операции и серверного валидатора. Если они объявлены как публичные, то могут быть вызваны из других операций. Переменные и параметры операции недоступны.
- Серверная валидация (код пишется на PL/Plus, выполняется на сервере БД или на сервере приложений) — выполняется в момент отрисовки ЭФО и в момент изменения состояния элементов управления ЭФО. В этом блоке доступны параметры и переменные, определённые на уровне всей операции. Используется для дополнительных проверок в зависимости от действий пользователя.
- Тело операции (код пишется на PL/Plus, выполняется на сервере БД или на сервере приложений) — выполняется после того, как пользователь нажал кнопку Ok на ЭФО. В этом блоке доступны параметры и переменные, определённые на уровне всей операции.
- Клиентская валидация (код пишется на VBS, выполняется в Навигаторе) — используется для изменения отображаемых элементов ЭФО в зависимости от действий пользователя. Доступны параметры и переменные операции, обращение к элементам ведётся по именам, заданным в редакторе ЭФО. Когда наследуется ЭФО, наследуется и клиентская валидация.

Для управления параметрами операции создаётся экранная форма (ЭФО). Расположение элементов и их настройка задаются в редакторе ЭФО. Элементы, отображаемые на ЭФО, могут переводится в различные режимы: только чтение, отключён, невидим.

Логически платформу развития можно разделить на 2 части: административную и пользовательскую. Каждой части соответствуют свои программные модули, называемые АРМами (Автоматизированное Рабочее Место пользователя).

Пользовательская часть — обеспечивает отражение модели системы, описанной администратором, в виде доступных пользователю информационных ресурсов. Пользовательская часть представлена АРМом «ЦФТ-Навигатор». АРМ может подключаться к БД напрямую, используя Oracle Client (двухуровневый способ соединения, 2L) или через Сервер Приложений 2МСА, используя HTTP-соединение (трёхуровневый способ соединения, 3L).

Административная часть — дает возможность описывать модель в терминах системы, регламентировать доступ пользователей к информационным ресурсам и выполнять функции текущего контроля состояния системы посредством доступа к журналам контроля работы пользователей.

Представлена она несколькими АРМами, из которых основные:

- АРМ «Администратор словаря данных» — предназначен для модификации модели данных: создания, изменения и удаления элементов модели;
- АРМ «Администратор проектов» — позволяет переносить элементы моделей данных между различными моделями, производить сравнение моделей;
- АРМ «Администратор доступа» — предназначен для формирования свойств, привилегий и прав доступа пользователей модели;
- АРМ «Рабочее место ревизора» — даёт возможность настраивать условия работы с системными журналами.

Использование АРМов «Администратор словаря данных» и «Администратор проектов» на данный момент заменено средой разработки CFT Platform IDE, о которой речь пойдёт более подробно в одном из следующих уроков. CFT Platform IDE также может выполнять и функции АРМа «Администратор доступа», но в менее наглядном варианте.

Присутствует и ряд вспомогательных АРМов административной части, имеющих более узкий спектр применения. Большинство из них предназначены для дополнительной настройки и обновления модулей платформы развития и администрирования модели в целом, а также для обработки выходных данных. Важным является вспомогательный АРМ «Монитор коммуникационного канала», который даёт возможность отслеживать действия запущенных в «ЦФТ-Навигатор» сессий и выводить отладочную информацию.

За основу информационной модели, с которой работают перечисленные АРМы, взята несколько изменённая классическая объектно-ориентированная модель. К ней применяется терминология, отличная от стандартной, как описано ранее. Она подразумевает наличие словаря, содержащего описание Классов (ТБП), которые характеризуются набором Атрибутов (реквизитов) и Методов (операций). Описание класса полностью определяет свойства и поведение Объектов – экземпляров данного Класса.

Основным АРМом для работы пользователей является «ЦФТ-Навигатор». Он является универсальным рабочим местом, позволяющим просматривать Экземпляры Типов Базовых Понятий (ТБП) и выполнять операции в соответствии с информационной моделью и правами доступа. Для того, чтобы появилась возможность работы с данными в АРМе «ЦФТ-Навигатор» необходимо:

1. Разработчику PL/Plus описать информационную модель
2. Администратору выдать права пользователю на соответствующие объекты: типы, представления, операции, переходы и т. д.

В системе «ЦФТ-Платформа развития» заложен многофункциональный пользовательский интерфейс, описанный далее. Это позволяет существенно упростить и ускорить разработку новых продуктов.

АРМ «ЦФТ-Навигатор» — это ПО, которое требует предварительной установки. Установка возможна только на рабочие станции с OC Windows. Установка может выполняться как на рабочее место пользователя, так и на терминальный сервер.

При запуске АРМа нам предлагается выбрать способ подключения:

1. Подключение через Сервер Приложений (3L);
2. Подключение через Oracle Client (2L);
3. Подключение через HTTP-сервер (3L).

В верхней части расположено главное меню, которое включает следующие пункты: «Управление», «Меню пользователя», «Окно», а также пункты, соответствующие доступным Базовым Понятиям (БП) системы, перечень которых определяется разработчиком системы.

В меню «Управление» вы можете настроить:

- количество одновременно запущенных экземпляров АРМа ЦФТ-Навигатор
- количество строк экземпляров отображаемых в представлении по умолчанию
- условия по умолчанию для регистра и даты при поиске экземпляров
- внешний вид АРМа

Под меню «Управление» расположена панель инструментов, которая включает в себя набор кнопок с пиктограммами, активация каждой из которых приводит к выполнению некоторых действий. Кнопки, доступные в активном окне, имеют насыщенный цвет, недоступные — приглушенный. Для каждой кнопки на панели инструментов задана всплывающая подсказка, кратко описывающая функцию кнопки.

В самом низу расположена строка состояния, в которую выводится такая информация, как наименование банка, пользователь, работающий в данный момент с системой, дата текущего операционного дня. Кроме того, в строке состояния может выводиться различная служебная информация, отражающая ход выполнения запросов и операций.

Здесь стоит упомянуть, что в процессе работы модуль «ЦФТ-Навигатор» может получать внешние сообщения. Такие сообщения содержат, например, информацию о произведенном изменении прав пользователя или об изменении структуры словаря данных. В особо критичных случаях может возникнуть сообщение, требующее закрытия АРМа. При получении такого сообщения, АРМ закрывается автоматически, предварительно уведомив об этом пользователя.

В меню «Управление» вы можете отключить отображение внешних сообщений.

В главном меню АРМа предлагается список элементов, соответствующих вершинам иерархии модели, сформированной разработчиком системы. Каждый пункт, соответствующий Базовому Понятию, позволяет выбрать из списка Тип Базового Понятия (ТБП), который в свою очередь может иметь экземпляры. При выборе одного из ТБП активизируется окно работы с ТБП. Это окно имеет заголовок, содержащий соответствующее наименование ТБП и текущее представление. При выборе ТБП сначала предлагается список его доступных представлений, если их количество больше одного. Если пользователю доступно только одно представление выбранного ТБП, то сразу показывается список его экземпляров. Если был предложен список представлений для выбора, то выбор из СУБД и визуализация списка экземпляров производится по двойному клику мыши на выбранном представлении либо по нажатию клавиши Enter.

Мы видим список экземпляров, давайте теперь узнаем, что мы можем с ним сделать:

1. Изменить ширину произвольного столбца. Для этого необходимо изменить ширину столбца в заголовке, захватив его границу мышью и перетащив в нужное положение.
2. Изменить порядок столбцов. Для этого необходимо захватить столбец в заголовке мышкой и перенести в нужную позицию.
3. Обновить список экземпляров (F5). Повторится запрос к БД.
4. Отсортировать данные по возрастанию или по убыванию по одному столбцу или по нескольким. Для сортировки по нескольким столбцам требуется зажать Ctrl и прокликать по столбцам в заголовке в необходимом порядке.
5. Суммирование по одной колонке или по нескольким. Для выделения нескольких столбцов также потребуется Ctrl и мышка.
6. Посчитать количество экземпляров (Ctrl+N).
7. Собрать статистику по всей колонке или только по выделенным строкам (Ctrl+I). Вычисляется общее количество экземпляров, их среднее, минимальное и максимальное значение, а также их сумма. Если тип колонки не поддерживает статистику, команда будет недоступна.
8. Выполнить контекстный поиск (Ctrl+F) экземпляра в текущей выборке по значению элементов выделенного столбца.
9. Установить фильтр (Alt+3). По команде вызывается окно формирования фильтра, который позволит сформировать выборку экземпляров, наложив условия на значения одного или нескольких реквизитов текущего представления. Если в окне фильтра есть строки, помеченные знаком *, одну из них обязательно надо заполнить. Крыж Дополнительно открывает окно, в которое можно ввести дополнительные условия поиска (1). Нажатие на кнопку Или добавляет колонки, в которые можно ввести значения, которые в запросе будут разделены оператором OR.
10. Копировать (Ctrl+C) значение текущей выделенной ячейки в буфер обмена.
11. Копировать значения выделенных строки (Ctrl+Shift+C) в буфер обмена.
12. Просмотреть (Ctrl+F3) значение выделенной ячейки в отдельном окне. Команда может быть полезна при просмотре объемных значений типа мемо-поле.
13. Выполнить поиск по обратным ссылкам (Ctrl+R). Команда реализует возможность просмотра экземпляров, ссылающихся значениями своих реквизитов на текущий экземпляр. Так как связь устанавливается через реквизит, то первоначально формируется иерархический список, в котором указываются: «Тип» — наименования ТБП, среди экземпляров которых найдены такие, значения реквизитов которых (указанных в столбце «Реквизит») являются ссылкой на текущий экземпляр.
14. Подняться по иерархии вверх или опуститься вниз. Команды позволяют изменить уровень иерархии в представлениях, содержащих иерархический реквизит. Если представление не имеет иерархического реквизита, данные команды недоступны.
15. Выполнить операции: конструкторы, редакторы, деструкторы и прочие. (2) Пометив c помощью кнопки Ctrl и мышки несколько записей, над ними можно вызвать списочную операцию.
16. Распечатать выбранные по условиям фильтра записи (Ctrl+P). При печати в файл, вы можете задать какие колонки представления будут выведен, задать их порядок, настроить печать заголовка таблицы и формат файла.
17. Вернуться с экземпляра к списку представлений можно с Alt+1
18. Посмотреть список реквизитов экземпляра — Alt+2. Если режим вызывается из списка представлений, то будет открыт список реквизитов первого экземпляра представления. При включенном режиме служебного интерфейса выводятся реквизиты ID — уникальный идентификатор записи, и уникальный идентификатор коллекции COLLECTION_ID (если запись принадлежит коллекции).
19. Просмотреть текст выполняемого запроса (Alt+6). Кнопка доступна при включенном режиме служебного интерфейса во всех режимах работы с ТБП: при просмотре: списка представлений, списка экземпляров и списка реквизитов экземпляра. Кнопка имеет разный вид в зависимости от платформы:

- При работе на «Платформе развития» — SQL
- При работе на «Платформе 2MCA» — XML

20. Справа вверху отображается ограничение количества строк, выдаваемых Навигатором по умолчанию (3). При необходимости двойным щелчком мыши по этой строке можно войти в режим редактирования количества и изменить значение.
21. Если ячейка является ссылкой на другой ТБП или ссылкой на массив (4), то двойным щелчком можно перейти к просмотру объектов, на которые она указывает.
22. Пиктограмма (5) покажет переходы, доступные для данного экземпляра.

CFT Platform IDE — это среда разработки проектов, основанных на ЦФТ-Платформе развития.

Поскольку исходный код на языке PL+ хранится и исполняется в базе данных Oracle, то остро встаёт вопрос о возможности параллельной разработки на этом языке и подключении современных систем хранения версий.

CFT Platform IDE позволяет содержать исходный код объектов модели в виде файлов на машине разработчика и в глобальном репозитории предпочитаемой системы хранения версия (как правило, используется git).

В процессе инициализации проекта необходимые элементы модели загружаются из БД на локальный компьютер разработчика.

Файлы исходного кода сохраняются в системе контроля версий, они же развертываются (deploy) в БД для последующего выполнения.

Одно из главных следствий наличия файлов исходного кода — изменение объектов БД только после выполнения процедуры развертывания (deploy) вне зависимости от масштабов произведенных изменений в файлах проекта.

Структура проекта.

Файлы исходного кода объектов модели находятся в папке `src`.

В структуре созданы папки:

- `ENTITY` — Базовые понятия
- `KERNEL` — Метаданные
- `TYPE` — Типы данных

`TYPE` может содержать подпапки базовых типов:

- `COLLECTION` — Коллекции (Массивы)
- `REFERENCE` — Ссылочные типы данных
- `STRING` — Строковые типы
- `NUMBER` — Числовые типы
- `VARIANT` — Варианты (Взаимоисключающие значения)
- `STRUCTURE` — Структуры и словари Далее следуют папки каждая из которых содержит файлы исходного кода объектов типа.

Вы всегда можете посмотреть на файлы проекта через Дерево типов.

Обратите внимание, что выполнение действий над папками означает выполнить указанное действие над всеми файлами (для которых данное действие применимо) во всех вложенных подпапках.

И наоборот, например, при возникновении ошибки компиляции декоратор Ошибка будет показан на всех папках содержащих объект имеющий ошибку.

Файлы объектов модели.

Кодировка файлов проекта может быть Cp1251, UTF-8.

`PLP` — исходный код операции.

```
class AC_FIN;

@name('Наименование операции')

method EDIT_AUTO is

    -- Секция Переменные

    @Name('V1') public V1 [STRING_2];

 

    -- Секция Локальные описания

    var2 string;

     

    public procedure p3 is begin null; end;

     

    function f2 return string is begin return null; end;

 

    -- Секции Параметры, Проверка, Тело

    EDIT_AUTO (

            @Name('Наименование параметра 1') p_name      [SBRF_NAME] := null,

            @Name('Наименование параметра 2') p_main_v_id [ACCNUM]    default null

        ) return [AC_FIN] is

     

        -- Проверка

        validate is

        begin

            if P_MESSAGE = 'DEFAULT' then

                p_name        := this.[NAME];

                p_main_v_id := this.[MAIN_V_ID];

            end if;

        end;

         

        -- Тело

        execute is

        begin

            [NAME]        := p_name;

            [MAIN_V_ID] := p_main_v_id;

        end;

     

    end; -- конец секций Параметры, Проверка, Тело

end; -- конец method EDIT_AUTO
```


Создание файлов ресурсов и изменение их кода в проекте в CFT Platform IDE не приводит к автоматическому появлению или изменению соответствующих объектов в базе данных. Для переноса модели в базу данных или ее изменения необходимо выполнить развертывание ресурсов.

В процессе развертывания ресурса обычно производится его блокирование в базе данных. После выполнения развертывания блокировку можно снять вручную или же установить настройку в IDE, по которой после выполнения развертывания блокировка будет сниматься автоматически. Также при редактировании существующий в базе данных объект можно заблокировать вручную, не дожидаясь выполнения развертывания.

Наглядное представление о синхронизации состояния ресурса в проекте с состоянием в базе данных, а также информацию о наличии/отсутствии блокировок ресурса дают декораторы.

### Декораторы

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%92%D0%B7%D0%B0%D0%B8%D0%BC%D0%BE%D0%B4%D0%B5%D0%B9%D1%81%D1%82%D0%B2%D0%B8%D0%B5-%D1%81-%D0%91%D0%94-\(Deploy-%D0%B2-%D0%91%D0%94\)#%D0%B4%D0%B5%D0%BA%D0%BE%D1%80%D0%B0%D1%82%D0%BE%D1%80%D1%8B)

Различают следующие декораторы:

- синхронизации состояния ресурсов проекта с базой данных,
- блокировки ресурсов в базе данных,
- ошибок и предупреждений.

Декораторы синхронизации состояния ресурсов

- состояние локального ресурса совпадает с тем, что хранится в БД, ресурс не содержит ошибок и не заблокирован.
- ресурс отсутствует в БД.

Данный декоратор устанавливается для файлов с расширением tbp, plp, vw.

Для файлов с расширениями vcs, vp, mcs, mp декоратор может быть установлен, если соответствующая операция или представление не существует в БД, либо для этого ресурса в базе данных не хранится информация о состоянии синхронизации. То есть для ресурса не выполнялось развертывание в БД.

- состояние локального ресурса отличается от того, которое хранится в БД.
- в БД отсутствует информация о состоянии синхронизации.
- в проекте отсутствует информация о состоянии синхронизации.

Такая ситуация возможна, если файл был добавлен в проект, при этом не выполнялась ни сборка проекта, ни сохранение файла, ни открытие его в редакторе; либо для ресурса нельзя получить его короткое имя вследствие того, что в исходном коде оно не задано или задано некорректно.

Данный декоратор устанавливается для файлов с расширением tbp, plp, vw. Для файлов с расширением vcs, vp, mcs, mp декоратор устанавливается временно в момент инициализации проекта, и затем сменяется одним из выше указанных в списке декораторов.

Декораторы блокировки ресурсов в базе данных

- ресурс заблокирован другим пользователем.
- ресурс заблокирован текущим пользователем.
- ресурс открыт на редактирование текущим пользователем в “Администраторe словаря данных”.

Декораторы ошибок и предупреждений

- ресурс содержит ошибки.
- ресурс содержит предупреждения.

Декораторы собственной блокировки и обозначений ошибок и предупреждений наследуются вверх по иерархии до корня проекта.

В процессе параллельной разработки приложения несколькими разработчиками возможны ситуации одновременного изменения одного и того же ресурса, в том числе с блокированием его в базе данных. Для анализа различий состояния ресурса в проекте и в базе данных следует использовать окно Deployment View.

### Команды, выполняющие развертывание

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%92%D0%B7%D0%B0%D0%B8%D0%BC%D0%BE%D0%B4%D0%B5%D0%B9%D1%81%D1%82%D0%B2%D0%B8%D0%B5-%D1%81-%D0%91%D0%94-\(Deploy-%D0%B2-%D0%91%D0%94\)#%D0%BA%D0%BE%D0%BC%D0%B0%D0%BD%D0%B4%D1%8B-%D0%B2%D1%8B%D0%BF%D0%BE%D0%BB%D0%BD%D1%8F%D1%8E%D1%89%D0%B8%D0%B5-%D1%80%D0%B0%D0%B7%D0%B2%D0%B5%D1%80%D1%82%D1%8B%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5)

- `Deploy` — выполняет развертывание выбранных ресурсов.
- `Deploy with recompile` — выполняет развертывание выбранных ресурсов с перекомпиляцией зависимых от них объектов.
- `Deploy locked` — выполняет развертывание всех ресурсов, заблокированных текущим пользователем.
- `Deploy VCS changed` — выполняет развертывание всех ресурсов, состояние которых отличается от состояния в системе контроля версий. Процесс развертывания не удаляет элементы модели (ТБП, операции, представления, индексы) со схемы БД, для удаления необходимо выполнить команду `Undeploy`.

Git — распределённая система контроля версий. Это означает, что кроме глобального удаленного репозитория на сервере каждый разработчик имеет у себя локальную копию репозитория, с которой и работает. Это в том числе позволяет выполнять коммиты даже если временно отсутствует соединение с сервером. Еще одним важным преимуществом Git является хорошо построенная работа с ветками и удобный механизм слияний.

Перед началом работы с Git необходимо выполнить настройку IDE, а также настроить профиль в используемом агрегаторе репозиториев, например, в Stash или в GitLab.

Работа с репозиторием производится в окне Git Repositories, по умолчанию открытом в соответствующей перспективе в CFT Platform IDE. Здесь можно работать со следующими элементами системы контроля версий:

- Branches — ветки проекта, извлеченные в рабочее пространство.
    - Local содержит локальную копию репозитория.
    - Remote Tracking содержит отслеживаемые ветки в удаленном репозитории.
- Tags — тэги проекта.
- References — ссылки на конкретные коммиты.

В системе контроля версий Git существует понятие индекса. Для того, чтобы изменения могли попасть в локальный репозиторий, они должны быть проиндексированы. В окне Git staging не проиндексированные файлы отображаются в поле Unstaged Changes. Если изменения были добавлены в индекс, а затем ресурс снова был изменен, то его необходимо добавить в индекс повторно. Добавление в индекс производится с помощью команды Add to Index или простым перетаскиванием из поля Unstaged Changes в поле Staged Changes.

В этом же окне с помощью кнопки Commit можно выполнить перенос изменений в локальный репозиторий. Также можно выполнить одновременный перенос в локальный и в удаленный репозитории по кнопке Commit and Push.

В окне History отображается информация обо всех изменениях, происходивших в репозитории. В этом окне можно посмотреть взаимосвязь между разными ветками, а также подробную историю коммитов. По каждому коммиту можно увидеть кем, когда и какие изменения были произведены. Из этого окна можно производить операции с коммитами, например, Reset, Revert commit или Cherry Pick.

Основные команды:

- Pull — забирает изменения из удаленного репозитория и производит слияние с активной веткой.
- Fetch — забирает изменения из удаленного репозитория.
- Create Branch — создает новую ветку от выбранной.
- Checkout — выполняет переключение между ветками, эта команда также вызывается по двойному клику на нужной ветке.
- Add to Index — добавляет содержимое рабочей директории в индекс для последующего коммита.
- Commit — переносит изменения из индекса в локальный репозиторий.
- Merge — выполняет слияние текущей и указанной веток, добавляя все изменения одним коммитом в момент слияния.
- Rebase — выполняет слияние текущей и указанной веток, встраивая каждый коммит вливаемой ветки в основную отдельно в хронологическом порядке.
- Cherry Pick — выполняет вливание отдельного выбранного коммита в текущую ветку.
- Push — переносит изменения из локального репозитория в удаленный.
- Reset — отменяет внесенные изменения. Существует три варианта этой команды:
    - Soft — указатель HEAD перемещается на выбранное состояние, оно становится текущим для локального репозитория, при этом внесенные изменения остаются в рабочем пространстве и в индексе.
    - Mixed — указатель HEAD перемещает на выбранное состояние, изменения остаются в рабочем пространстве, но удаляются из индекса.
    - Hard — указатель HEAD перемещает на выбранное состояние, изменения удаляются из рабочего пространства.
- Revert commit — создает коммит, полностью обратный последнему выполненному коммиту.

Файлы в репозитории могут находиться в следующих состояниях:

- отслеживаемый файл (tracked). Ресурс известен Git и контролируется.
- не отслеживаемый файл (untracked). Ресурс неизвестен Git и не будет контролироваться, пока не будет явно добавлен в область отслеживаемых файлов с помощью команды Add.
- игнорируемый файл (ignored). Ресурс игнорируется Git. Для добавления ресурса в игнорируемые, информацию о нем нужно занести в файл .gitignore, находящийся в проекте. Также игнорируемые ресурсы можно указать в окне настроек Window — Preferences — Team — Ignored Resources. В игнорируемые ресурсы рекомендуется добавлять производные, генерируемые файлы, не представляющие собой исходный код.
- измененный файл (dirty). Ресурс содержит изменения, не добавленные в индекс.
- подготовленный файл (staged). Ресурс содержит изменения, которые были добавлены в индекс.
- частично подготовленный файл (partially-staged). Ресурс был изменен и добавлен в индекс, после этого в файл были внесены другие изменения (соответственно, необходимо снова выполнить добавление в индекс).
- добавленный файл (added). Новый ресурс, который не известен Git, но уже добавлен в индекс.
- удаленный файл (removed). Ресурс подготовлен для удаления из Git.
- конфликтный файл (conflicted). Файл содержит изменения, конфликтующие со сравниваемой веткой.

Возможный базовый сценарий работы над задачей:

1. Создать ветку для работы над задачей от основной ветки master с помощью команды Create Branch.
2. Внести необходимые изменения по задаче.
3. Добавить изменения в индекс командой Add to Index.
4. Залить изменения в локальный репозиторий с помощью команды Commit.
5. Добавить в ветку задачи изменения, произошедшие в основной ветке за время работы над задачей. Воспользоваться для этого командой Merge.
6. Разрешить возникшие конфликты.
7. Переключиться на основную ветку двойным кликом по ней или с помощью команды Checkout.
8. Выполнить слияние изменений из ветки по задаче в основную ветку командой Merge.
9. Залить изменения в удаленный репозиторий при помощи команды Push.

Для создания операций необходим специализированный язык программирования, отвечающий следующим требованиям:

- типы базовых понятий (ТБП) и их экземпляры словаря данных системы должны быть доступны наряду с предопределенными типами языка;
- вся структура хранения данных должна быть скрыта, чтобы при ее изменении не потребовалось переписывать операции;
- язык следует реализовать в виде конвертера, что позволит выбирать наиболее эффективный для конкретной платформы выходной язык;
- язык должен быть достаточно простым, с небольшим количеством традиционных управляющих структур, это позволит, в частности, иметь в качестве выходного языка практически любой язык программирования и, при необходимости, перейти к трехуровневой модели.

Так как в текущей реализации операции выполняются на сервере Oracle, то был разработан препроцессор, создающий текст операций на языке хранимых процедур PL/SQL, а сам язык спроектирован как «надмножество подмножества PL/SQL». Его условное название — PL/Plus. Для упрощения работы с экземплярами ТБП на уровне языка PL/SQL, который ничего не знает о внутренних свойствах ТБП, для каждого ТБП создается так называемый интерфейсный пакет типа, который содержит в себе набор низкоуровневых базовых функций и процедур работы с экземплярами ТБП (создание, удаление экземпляров, чтение и изменение значений реквизитов и другие). Препроцессор PL/Plus элементарные действия с экземплярами ТБП преобразует в PL/SQL вызовы базовых процедур интерфейсных пакетов типов, более сложные конструкции (аналоги SQL-операторов) преобразуются непосредственно в SQL-конструкции языка PL/SQL.

В данном документе будут описаны, в основном, только возможности PL/Plus по манипуляции данными, описанными в словаре системы, остальные сведения можно найти в описании PL/SQL и SQL базы данных (БД) Oracle (более того, документ частично использует терминологию БД Oracle в случаях, когда нет перекрытых терминов в PL/Plus).

В языке PL/Plus используются предопределенные типы данных, которые можно разделить на несколько групп:

- числовые (number, integer, binary_float, binary_double)
- символьные (string, clob, nstring)
- логические (boolean)
- время (date, timestamp)
- бинарные (raw, blob)
- null

### Числовые типы

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%91%D0%B0%D0%B7%D0%BE%D0%B2%D1%8B%D0%B5-%D1%82%D0%B8%D0%BF%D1%8B-%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85#%D1%87%D0%B8%D1%81%D0%BB%D0%BE%D0%B2%D1%8B%D0%B5-%D1%82%D0%B8%D0%BF%D1%8B)

Основным числовым типом, заимствованным из СУБД Oracle является тип `number`. Это тип для работы с целыми числами, а также с числами с фиксированной и плавающей точкой. Имеет очень широкий диапазон допустимых значений. Позволяет указать количество значащих разрядов и количество знаков после запятой. Значения могут лежать в диапазоне от 10 в минус 84-й степени до 10 в 127-й. Синтаксис следующий: number, в скобках количество разрядов, затем количество знаков после запятой.

Важной особенностью является отсутствие ошибок округления, присущих «машинным» типам с плавающей запятой, а также независимость от платформ и операционных систем, на которых выполняются такие расчеты — на любой платформе результаты вычислений с одними и теми же значениями, представленными типом number будут одинаковыми. Эта особенность делает тип number незаменимым для расчета финансовых величин, таких как сумма и проценты.

Допускается использование типа без указания точности — number. Такое объявление позволяет использовать как целые, так и дробные значения. При указании только точности количество знаков после запятой принимается равным 0, объявление `number(10)` эквивалентно `number(10, 0)`. При ограниченном количестве знаков после запятой округление производится по арифметическим правилам.

Примеры:

```
a number := 123;
b number(12, 2) := 123.45;
c number := -890.4575757;
d number(38, 6) := 123456.789E-3;
e number(10) := 123;
```

### Целочисленный тип `integer`

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%91%D0%B0%D0%B7%D0%BE%D0%B2%D1%8B%D0%B5-%D1%82%D0%B8%D0%BF%D1%8B-%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85#%D1%86%D0%B5%D0%BB%D0%BE%D1%87%D0%B8%D1%81%D0%BB%D0%B5%D0%BD%D0%BD%D1%8B%D0%B9-%D1%82%D0%B8%D0%BF-integer)

Позволяет хранить целые значения от минус двух до двух миллиардов. Такой диапазон относительно невелик и намного уступает возможностям `number`, но `integer` является «машинным« типом. То есть типом, операции с которым выполняются процессорами напрямую. Поэтому `integer` намного быстрее `number`. Если решаемая задача предусматривает использование только целых чисел и позволяет ограничиться диапазоном `integer`, то предпочтительнее использовать `integer`, а не `number`. Важно, что `integer` языка PL+ не соответствует типу `integer` Oracle PL/SQL, который является целочисленным вариантом `number`. Тип `integer` в PL+ соответствует `pls_integer` (binary_integer) в Oracle.

Примеры:

```
a integer := 1234;
b integer := 32767;
min_int integer := -2147483648;
max_int integer := 2147483647;
```

### Тип с плавающей точкой

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%91%D0%B0%D0%B7%D0%BE%D0%B2%D1%8B%D0%B5-%D1%82%D0%B8%D0%BF%D1%8B-%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85#%D1%82%D0%B8%D0%BF-%D1%81-%D0%BF%D0%BB%D0%B0%D0%B2%D0%B0%D1%8E%D1%89%D0%B5%D0%B9-%D1%82%D0%BE%D1%87%D0%BA%D0%BE%D0%B9)

Тип с плавающей точкой (или плавающей запятой), стандартной (`binary_float`) и двойной (`binary_double`) точности. Также являются «машинными», что дает преимущество в скорости вычислений. Однако, как и в большинстве реализаций и языков, операции вычисления с этими типами могут накапливать ошибки округления, вызванные ограничением размера хранения и несоответствием основания систем счисления. По вышеизложенным причинам, типы с плавающей точкой не подходят для финансовых расчетов, но могут использоваться для промежуточных вычислений с последующим округлением до заданной точности, а также в физических и математических расчетах, где точность чисел в десятичном представлении не так важна.

Примеры:

```
a binary_float := 123.456F;
b binary_double := 4.789456e9D;
```

### Символьные типы

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%91%D0%B0%D0%B7%D0%BE%D0%B2%D1%8B%D0%B5-%D1%82%D0%B8%D0%BF%D1%8B-%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85#%D1%81%D0%B8%D0%BC%D0%B2%D0%BE%D0%BB%D1%8C%D0%BD%D1%8B%D0%B5-%D1%82%D0%B8%D0%BF%D1%8B)

Следующий класс — символьные типы. Итак, `string`. Пожалуй, самый распространенный тип данных, если не по количеству значений, то по использованию памяти и дискового пространства. Тип `string` в PL+ полностью соответствует `varchar2` в Oracle. Допустимая максимальная длина строки до 32767 символов. Тип `string` может использоваться с указанием максимальной длины строки: `string(100)`. Это означает, что в переменная с типом `string(200)` может содержать строки длиной до 200 символов. При попытке сохранить в переменную строку длиннее будет вызвано исключение. Кодировка символов в строках устанавливается в настройках БД и является единой для всех таблиц БД и переменных программного кода. PL+ позволяет не указывать максимальную длину строки после `string`. Такое объявление эквивалентно объявлению `string(128)`. Также PL+ допускает использование описания `varchar2` вместо `string`. Никакой разницы между ними нет.

Примеры:

```
a string := 'AaBbCc';
b string(50) := 'It''s a string!';
c string(200) := '... конец одной строки
начало другой строки ...
    а эта строка началась с табуляции';
e varchar2 := 'Нет отличия от string';
```

Тип `nstring` очень похож на `string`. Все аналогично, но кодировка строки не зависит от БД и хранится в Unicode. Максимальная длина указывается в символах, а не в байтах. Ограничение на 32Kb сохраняется, поэтому максимально возможная длина `nstring` — 16383.

Для работы с большими символьными массивами, до 4Gb, используется тип `clob`. По сравнению со строками, имеет более сложную структуру, работает медленнее, а для обращения к нему часто необходимо использовать служебную библиотеку Oracle. Использовать `clob` нужно только тогда, когда действительно нельзя ограничиться 32Kb текста в переменных в коде или когда требуется хранить в БД строки более 4000 символов.

### Логический тип

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%91%D0%B0%D0%B7%D0%BE%D0%B2%D1%8B%D0%B5-%D1%82%D0%B8%D0%BF%D1%8B-%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85#%D0%BB%D0%BE%D0%B3%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B8%D0%B9-%D1%82%D0%B8%D0%BF)

Логический тип представлен одним видом данных — `boolean`.

Предназначен для хранения логических значений. Это тип логических выражений, используемых в операторах `if`, `while` или в секции `where` выражения `select`. Литералы логического типа имеют всего два значения: `true` и `false`. Важно не забывать, что переменная этого типа (как и любого другого) может принимать значение `null`.

### Тип для хранения дат и времени

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%91%D0%B0%D0%B7%D0%BE%D0%B2%D1%8B%D0%B5-%D1%82%D0%B8%D0%BF%D1%8B-%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85#%D1%82%D0%B8%D0%BF-%D0%B4%D0%BB%D1%8F-%D1%85%D1%80%D0%B0%D0%BD%D0%B5%D0%BD%D0%B8%D1%8F-%D0%B4%D0%B0%D1%82-%D0%B8-%D0%B2%D1%80%D0%B5%D0%BC%D0%B5%D0%BD%D0%B8)

Типами для хранения дат и времени являются `date` и `timestamp`. Максимальная точность времени типа `date` — секунды. Этот тип напрямую несовместим с числовыми значениями, но связан с ними. Например если вычесть из одной даты другую дату, то получится числовое значение, причем, целая часть будет содержать количество полных дней между датами, а дробная часть — время. Подробнее о манипуляциях с `date` поговорим чуть позже. `timestamp` аналогичен `date`, но с более высокой точностью. Точность `timestamp` достигает микросекунд на некоторых платформах. Точность может быть больше или меньше в зависимости от той точности, которую предоставляет ОС. Тип `timestamp` совместим с `date`.

### Хранение цифровых данных

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%91%D0%B0%D0%B7%D0%BE%D0%B2%D1%8B%D0%B5-%D1%82%D0%B8%D0%BF%D1%8B-%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85#%D1%85%D1%80%D0%B0%D0%BD%D0%B5%D0%BD%D0%B8%D0%B5-%D1%86%D0%B8%D1%84%D1%80%D0%BE%D0%B2%D1%8B%D1%85-%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85)

Для хранения цифровых данных неопределенного в БД типа существуют типы, которые представляют собой по сути массивы байт, так называемые бинарные типы. Первый из них, `raw`, во многом похож на `string`, у которого вместо символов — байты. При объявлении переменных типа `raw` также указывается максимальная длина. Например, `bin raw(100);` Также как у `string`, максимальная длина `raw` не может превышать 32767 байта. А в БД не может храниться `raw` больше 4000 длиной, опять же, как `string`. Значение типа `raw` может быть приведено к строке, при этом получится строка, содержащая HEX-представлений байт, лежащих в `raw`. Для некоторых манипуляций и приведения к другим типам есть специальная библиотека `utl_raw`.

### Тип `blob`

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%91%D0%B0%D0%B7%D0%BE%D0%B2%D1%8B%D0%B5-%D1%82%D0%B8%D0%BF%D1%8B-%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85#%D1%82%D0%B8%D0%BF-blob)

Тип `blob` используется как хранилище огромных массивов байт, почти неограниченного размера. Как правило, используется для хранения содержимого файлов произвольного формата. Также это может быть текст в кодировке, отличной от кодировки БД и поэтому несовместимый непосредственно со строковыми типами. По способу доступа и методам обработки похож на `clob`. И также, если `clob` — это большой `string`, то `blob` — большой `raw`. Для работы с `blob` и `clob` существует библиотека `dbms_lob`.

Примеры:

```
function file2blob(p_filedir varchar2, p_filename varchar2) return BLOB is
v_blob BLOB;
file integer;
amount integer := 32767;
offset integer := 1;
buffer raw(32767);
cnt integer;
begin
dbms_lob.createtemporary(v_blob, true, dbms_lob.call);
file := stdio.f_open(p_filedir||'/'||p_filename, 'r');
if file < 0 then
     pragma error('Ошибка при загрузке файла: '||file||' - '||stdio.error_message(file));
end if;
begin
     loop
          cnt := stdio.f_read(file, buffer, amount);
          exit when cnt <= 0;
          if buffer is not null then
               dbms_lob.write(v_blob, cnt, offset, buffer);
               offset := offset + cnt;
          end if;
     end loop;
exception when others then null;
end;
cnt := stdio.f_close(file);
return v_blob;
end;
```

### `NULL`

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%91%D0%B0%D0%B7%D0%BE%D0%B2%D1%8B%D0%B5-%D1%82%D0%B8%D0%BF%D1%8B-%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85#null)

Есть особое значение, которое используется для всех типов в переменных, параметрах в коде и в таблицах БД. Это `null`. Интерпретировать это значение можно как отсутствие значения или неопределенное значение. В выражениях `null` может неожиданным образом влиять на результат. Чтобы избежать неожиданностей значение `null` нужно правильно обрабатывать и учитывать. Для проверки на `null` используется не сравнение (=), а оператор `is`. Например, `v is null`. Для строкового типа значение `null` неотличимо от пустой строки, так логическое выражение `'' is null` истинно. Любое сравнение с `null` даст результат `false`, как мы видим на примере, даже если значение переменной отлично от `null`, все равно проверка на неравенство даст отрицательный результат. В таблицах БД значения `null` не хранятся и не занимают место.

Примеры:

```
a boolean := true;
begin
    if a != null then
        stdio.put_line_buf('Сюда мы не попадём, хотя true отличается от null!');
    else
        stdio.put_line_buf('Мы попадём сюда!');
    end if;
    if '' is null then
        stdio.put_line_buf('Пустая строка - это null, сюда мы тоже попадём!');
    end if;
end;
```

В PL/Plus переменные условно можно разделить на переменные операции и все остальные переменные. Переменные операции относятся к глобальным переменным, но в отличии от обычных глобальных переменных могут использоваться для отображения контролов на экранной форме операции. Переменные операции объявляются в начале операции, синтаксис для объявления переменных операции выглядит так:

```
@name('Наименование переменной операции') public V_VAR [BOOLEAN];
```

Для объявления всех остальных переменных используется синтаксис:

```
[public] имя [constant] тип_переменной [:= умолчательное_значение_переменной];
```

Ключевое слово `public` указывает на то, что переменная является глобальной переменной операции. Ключевое слово `constant` указывает на то, что переменная является константой, то есть её значение не может быть изменено после инициализации. Если при инициализации переменной требуется присвоить какое-то значение, то это значение указывается после указания типа переменной и символа присваивания `:=`. Также отмечу, что в PL+ присутствуют переменные, которые не требуют явного определения — переменные циклов.

### Область видимости

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%9F%D0%B5%D1%80%D0%B5%D0%BC%D0%B5%D0%BD%D0%BD%D1%8B%D0%B5#%D0%BE%D0%B1%D0%BB%D0%B0%D1%81%D1%82%D1%8C-%D0%B2%D0%B8%D0%B4%D0%B8%D0%BC%D0%BE%D1%81%D1%82%D0%B8)

Важным свойством переменных является область видимости. Данное свойство определяет, можно ли вообще обращаться к этим переменным, можно ли обращаться к ним только по имени или же к имени необходимо добавлять какие-то префиксы.

### Срок жизни

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%9F%D0%B5%D1%80%D0%B5%D0%BC%D0%B5%D0%BD%D0%BD%D1%8B%D0%B5#%D1%81%D1%80%D0%BE%D0%BA-%D0%B6%D0%B8%D0%B7%D0%BD%D0%B8)

Следующим не менее важным свойством переменных является срок их жизни. Данное свойство определяет, когда переменная инициализируется и когда высвобождается зарезервированная под неё память.

### Глобальные переменные операций

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%9F%D0%B5%D1%80%D0%B5%D0%BC%D0%B5%D0%BD%D0%BD%D1%8B%D0%B5#%D0%B3%D0%BB%D0%BE%D0%B1%D0%B0%D0%BB%D1%8C%D0%BD%D1%8B%D0%B5-%D0%BF%D0%B5%D1%80%D0%B5%D0%BC%D0%B5%D0%BD%D0%BD%D1%8B%D0%B5-%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D0%B9)

Глобальные переменные объявляются на уровне операций (библиотек, групповых операций, простых операций и т.д.) и могут быть объявлены двумя способами, о которых упоминалось ранее. К таким переменным можно обращаться из любой части любой операции. Но есть особенности в формате обращения:

1. Если обращение выполняется из той же операции, где сами объекты объявлены, то обращаться к объектам можно без использования каких-либо префиксов
2. Если обращение будет выполняться из иной операции, то обращаться к объектам можно только с явным указанием операции в виде

```
::[CLASS].[SHORTNAME].var
```

### Локальные переменные операции

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%9F%D0%B5%D1%80%D0%B5%D0%BC%D0%B5%D0%BD%D0%BD%D1%8B%D0%B5#%D0%BB%D0%BE%D0%BA%D0%B0%D0%BB%D1%8C%D0%BD%D1%8B%D0%B5-%D0%BF%D0%B5%D1%80%D0%B5%D0%BC%D0%B5%D0%BD%D0%BD%D1%8B%D0%B5-%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D0%B8)

Локальные переменные операции так же объявляются в операциях (библиотек, групповых операций, простых операций и т.д.), но без использования ключевого слова `public`. Область видимости локальных переменных операции ограничивается самой операцией. Это означает что к этим переменным можно обращаться только из текущей операции. Эти переменные объявляются только для использования внутри самой операции: выделение переменных для кэширования или обмена данными между разными вызовами подпрограмм. При обращении к локальным переменным не указывается никаких префиксов.

Срок жизни глобальных и локальных переменных операций ограничен сроком жизни сессии. Инициализация таких переменных выполняется при первом любом обращении к операции: вызов функции или процедуры или обращение к любой глобальной переменной. После этого значение переменной самостоятельно не очищается на протяжении всего времени жизни сессии. Это очень удобно, например, для какого-нибудь кэширования данных или передачи данных между вызовами различных подпрограмм в рамках одной транзакции или даже в рамках разных. Важно помнить, если эти переменные используются для кэширования, то вам самостоятельно надо контролировать все моменты, когда требуется выполнять сброс кэша, то есть очистка переменных. Также важно помнить, что при возникновении исключительной ситуации (exception) сброс значений переменных операции также не выполняется.

### Локальные переменные функций или процедур

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%9F%D0%B5%D1%80%D0%B5%D0%BC%D0%B5%D0%BD%D0%BD%D1%8B%D0%B5#%D0%BB%D0%BE%D0%BA%D0%B0%D0%BB%D1%8C%D0%BD%D1%8B%D0%B5-%D0%BF%D0%B5%D1%80%D0%B5%D0%BC%D0%B5%D0%BD%D0%BD%D1%8B%D0%B5-%D1%84%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D0%B9-%D0%B8%D0%BB%D0%B8-%D0%BF%D1%80%D0%BE%D1%86%D0%B5%D0%B4%D1%83%D1%80)

Если требуется объявить переменные, которые будут доступны только в конкретной функции или процедуре, то для этого их необходимо объявить в блоке объявления переменных соответствующей функции или процедуры. В этом случае видимость этих переменных будет ограничиваться только текущей подпрограммой. Важно заметить, если функция объявлена внутри функции (далее вложенная функция), то область видимости переменных основной функции распространяется на вложенную функцию, то есть внутри вложенной функции можно обращаться к переменным основной функции. Однако обратное неверно, то есть область видимости переменных вложенных функций ограничивается самими этими вложенными функциями.

Срок жизни локальных переменных функций и процедур ограничен вызовом соответствующей функции или процедуры. При этом инициализация переменных выполняется в самом начале до исполнения тела функции, а сброс — по завершении функции (как успешному, так и нет). Если речь идёт про вложенные функции и процедуры, то срок жизни переменных вычисляется аналогично, то есть он ограничен вызовом соответствующей вложенной функции или процедуры. Из этого следует, что при каждом вызове вложенной функции её переменные будут заново инициализироваться, даже если эти вызовы выполняются в рамках одного вызова основной функции.

А можно ли объявить переменную так, чтобы её область видимости была меньше чем полный текст функции?

Да можно. Для этого необходимо внутри функции воспользоваться конструкцией

```
declare
    ...
begin
    ...
end;
```

и объявить переменную в блоке `declare`. В этом случае область видимости этой переменной будет ограничена текущим блоком `declare ... begin ... end`, и срок жизни так же будет ограничен этим же блоком `declare ... begin ... end`.

Переменные, которые объявлены в цикле по курсору или по интервалу

Область видимости переменных, которые объявлены в цикле по курсору или по интервалу, ограничена самим циклом. Таким образом нет возможности обращаться к этим переменным вне этого цикла. Срок жизни таких переменных, так же ограничен самим циклом.

А как себя поведёт СУБД, если в рамках одной области видимости будет видно несколько переменных с одинаковым наименованием?

Лучше так не делать, но что если это уже кто-то сделал до вас, как себя поведёт СУБД? СУБД не позволить на одном уровне создать несколько переменные с одинаковыми наименованиями, но она это позволит сделать, если уровни будут разливаться. Например, нельзя создать 2 локальные переменные операции с одинаковым наименованием, но можно создать локальную переменную операции и локальную переменную функции этой же операции с аналогичным наименованием. Таким образом внутри функции пересекаются (если быть точным, то вкладываются) области видимости этих переменных. В этом случае СУБД отдаст предпочтение переменным, которые имеют более узкую область видимости, то есть при обращении к переменной по наименованию будет отдано предпочтение локальной переменной функции перед локальной переменной пакета.

Выражение — это конструкция языка программирования, состоящая из констант, переменных (операндов) и знаков операций (операторов). Выражение описывает алгоритм вычисления некоторого значения.

Большинство операторов предназначены для двух значений, между которыми они указаны. Но есть несколько операторов, которые указываются перед одним операндом, на значение которого и влияют. Такие операторы называются унарными, т.е. работающими с одним операндом.

В зависимости от типов используемых значений и операторов, значение выражения может быть логическим, числовым, строковым или датой.

### Логические выражения

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%92%D1%8B%D1%80%D0%B0%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F-%D0%B8-%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%BE%D1%80%D1%8B#%D0%BB%D0%BE%D0%B3%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B8%D0%B5-%D0%B2%D1%8B%D1%80%D0%B0%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F)

Такие выражения принимают значения истина (true) или ложь (false). Логические выражения используются в условных конструкциях, в циклах и просто как значения переменных, параметров, результатов функций. Логические выражения не обязательно состоят только из операндов, имеющих логические значения. Могут использоваться числовые, строковые и другие простые (скалярные) значения, в том числе и значения выражений. В сочетании с операторами сравнения, значения других типов дают логический результат.

|Операторы|Название|Описание|
|---|---|---|
|=|равенство|Возвращает истину, если значения операндов равны|
|!=, <>|не равно|Возвращает истину, если значения операндов отличаются|
|<, >, <=, >=|строгие и нестрогие неравенства|Возвращают истину, если значение операнда слева меньше, больше, меньше или равно, больше или равно значения операнда справа, соответственно|
|IS NULL|проверка на NULL|Возвращает истину, если операнд слева имеет значение NULL|
|IS NOT NULL|проверка на NULL|Возвращает истину, если операнд слева имеет значение отличное от NULL|
|LIKE|соответствие строки шаблону|Возвращает истину, если строковое значение слева соответствует шаблону справа|
|IN|равен одному из|Возвращает истину, если значение слева равно хотя бы одному из значений справа, перечисленных в скобках через запятую|
|NOT IN|не равен ни одному из|Возвращает истину, если значение слева не равно ни одному из значений справа, перечисленных в скобках через запятую|

Мы видим список существующих операторов. Во первых, это группа проверки операндов на равенство, неравенство, отношения меньше, больше, меньше или равно, больше или равно. Затем идёт проверка на соответствие операнда значению NULL — `IS NULL` и `IS NOT NULL`. Оператор `LIKE` проверяет строковое значение на соответствие шаблону. Подробнее о шаблонах `LIKE` речь пойдет в строковых выражениях. Операторы `IN` и `NOT IN` проверяют вхождение значения в указанный список.

### Операторы для числовых выражений

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%92%D1%8B%D1%80%D0%B0%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F-%D0%B8-%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%BE%D1%80%D1%8B#%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%BE%D1%80%D1%8B-%D0%B4%D0%BB%D1%8F-%D1%87%D0%B8%D1%81%D0%BB%D0%BE%D0%B2%D1%8B%D1%85-%D0%B2%D1%8B%D1%80%D0%B0%D0%B6%D0%B5%D0%BD%D0%B8%D0%B9)

Теперь у нас на экране знакомые со школы операторы для числовых выражений, они же арифметические. В числовых выражениях используются операнды, имеющие числовое значение и арифметические операторы. Приоритет выполнения также совпадает с арифметическими операциями.

|Операторы|Название|Описание|
|---|---|---|
|+|сложение|Арифметическое сложение операндов слева и справа|
|-|вычитание|Вычитает из операнда слева операнда справа|
|*|умножение|Умножает операнд слева на операнд справа|
|/|деление|Делит операнд слева на операнд справа|
|**|возведение в степень|Возводит значение слева в степень со значением справа|
|-|унарный минус|Меняет знак операнда справа на противоположный. Эквивалентно умножению на -1|

Оператор манипуляции со строками

```
hello := 'Hello,' || ' ' || 'world!';
news := 'Сегодня ' || sysdate || ' и это будет прекрасный день!';
song := 'Жили у ' || grandmom || ' ' || goose_count || ' веселых гуся. Один - ' || goose_1_color || ', другой - ' || goose_2_color || ', ' || goose_count || ' веселых гуся.';
```

Главный (и единственный) оператор манипуляции со строками — оператор конкатенации. Он «склеивает» в одну строку строки, между которыми находится. Также существует множество функций для работы со строками, мы их рассмотрим на следующем уроке.

В этом примере мы ввели и оператор присваивания, выглядит как двоеточие равно. Он позволяет присвоить переменной слева от оператора (например, hello) значение выражения, указанного справа - Hello world.

### Сравнение строк

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%92%D1%8B%D1%80%D0%B0%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F-%D0%B8-%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%BE%D1%80%D1%8B#%D1%81%D1%80%D0%B0%D0%B2%D0%BD%D0%B5%D0%BD%D0%B8%D0%B5-%D1%81%D1%82%D1%80%D0%BE%D0%BA)

Для сравнения строк могут использоваться те же операторы, что и для числовых значений. С равенством все просто — сравнение двух строк вернет истину, только если строки содержат одинаковые последовательности символов. Но когда одна строка больше другой? В случае, если одна строка начинается с другой строки, больше та строка, что длиннее. 'ABC' меньше, чем 'ABCdef'. Вообще, если начало строк совпадает, а продолжение разное, то начало можно не учитывать — результат сравнения будет таким же как сравнение продолжений: 'ABCklm' < 'ABCxyz', потому что 'k' меньше 'x'. То есть при сравнении строк сравнение продолжается до тех пор, пока одна из строк не кончится, или символы строк в соответствующих позициях будут не равны. В этом случае результатом сравнения строк будет результат сравнения этих двух символов. Сравнение символов происходит в соответствии с числовыми значениями их кодов.

```
flag1 := 'ABC' = 'aBC'; /* false */
flag2 := 'ABC' = 'ABC'; /* true */
flag3 := 'ABC' > '123'; /* true */
flag4 := 'ABC' > 'abc'; /* false */
flag5 := 'ABC1' < 'ABC2'; /* true */
flag6 := 'AB_Z' < 'ABC'; /* false (_ больше C) */
flag7 := 'AB_Z' < 'ABc'; /* true (c больше _) */
```

Для сравнения строк используется еще один важный оператор — `LIKE`. Он проверяет строку на соответствие шаблону. Шаблоном является строковое выражение, содержащее подстановочные символы, т. е. символы, которые обозначают другой символ или несколько других символов. Символ `%` в шаблоне обозначает любое количество любых символов, а символ `_` обозначает ровно один любой символ в строке. Шаблон 'cat%' соответствует всем строкам, начинающимся с 'cat', шаблон '%pig' — строкам, заканчивающимся на 'pig', а '%bear%' — строкам, содержащим 'bear' в любом месте.

```
flag1 := 'police' like 'poli%'; /* true */
flag2 := 'megapolis' like 'poli%'; /* false */
flag3 := 'hopeless' like '%less'; /* true */
flag4 := 'happy' like '%less'; /* false */
flag5 := 'for' like '%for%'; /* true */
flag6 := 'fortune' like '%for%'; /* true */
flag7 := 'comfortable' like '%for%'; /* true */
flag8 := 'phosphorus' like '%for%'; /* false */
flag9 := 'go for' like '%for%'; /* true */

flag10 := 'mega' like 'me_a'; /* true */
flag11 := 'meta' like 'me_a'; /* true */
flag12 := 'mem' like 'me_a'; /* false */
flag13 := 'me' like 'me_'; /* false */
```

### Выражения с датами

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%92%D1%8B%D1%80%D0%B0%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F-%D0%B8-%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%BE%D1%80%D1%8B#%D0%B2%D1%8B%D1%80%D0%B0%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F-%D1%81-%D0%B4%D0%B0%D1%82%D0%B0%D0%BC%D0%B8)

Значения, отражающие дату и время, также могут участвовать в выражениях. С датами допустимы операции сложения и вычитания, но с ограничениями. К дате можно прибавить или вычесть число, результатом будет дата, отстоящая от первоначальной на количество суток, указанное в числе. Разность двух дат дает числовой интервал между ними. Дробная часть числа отвечает за время, например одна вторая — это половина суток, то есть 12 часов. Сложение дат смысла не имеет.

```
tomorrow := today + 1; /* завтра */
expiration := poduction + 1.5; /* 36 часов */
five_seconds := sysdate + 5 / 86400; /* через 5 секунд */
two_weeks_ago := now - 14; /* 2 недели назад */
days := date_end - date_start; /* продолжительность, дней */
```

### Применение операторов с пустой строкой

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%92%D1%8B%D1%80%D0%B0%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F-%D0%B8-%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%BE%D1%80%D1%8B#%D0%BF%D1%80%D0%B8%D0%BC%D0%B5%D0%BD%D0%B5%D0%BD%D0%B8%D0%B5-%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%BE%D1%80%D0%BE%D0%B2-%D1%81-%D0%BF%D1%83%D1%81%D1%82%D0%BE%D0%B9-%D1%81%D1%82%D1%80%D0%BE%D0%BA%D0%BE%D0%B9)

Рассмотрим подробнее применение операторов с пустой строкой, которая является в PL+ особым значением, `null`. Если мы «приклеим» к строке `null`, значение строки не изменится.

Все арифметические операции между `null` и датой или числом будут иметь результат — неопределенность, `null`.

Любой оператор равенства, неравенства и сравнения любых значений с `null` дает `null`. А отрицание неопределенности `not null` все равно неопределенность. Часто это вызывает ошибки в условных конструкциях, где используются логические выражения. Чтобы понимать как поведет себя программа в этих случаях нужно запомнить, что условие считается выполненным только если его результат `true`. В остальных случаях условие не выполняется или цикл не продолжается.

```
flag1 := 'ABC' = ('ABC' || ''); /* true */
flag2 := 1 + NULL; /* null */
flag3 := 'ABC' > NULL; /* null */
flag4:= not (123 > NULL); /* null */
```

### Правила вычисления логических выражений c `null`

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%92%D1%8B%D1%80%D0%B0%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F-%D0%B8-%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%BE%D1%80%D1%8B#%D0%BF%D1%80%D0%B0%D0%B2%D0%B8%D0%BB%D0%B0-%D0%B2%D1%8B%D1%87%D0%B8%D1%81%D0%BB%D0%B5%D0%BD%D0%B8%D1%8F-%D0%BB%D0%BE%D0%B3%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B8%D1%85-%D0%B2%D1%8B%D1%80%D0%B0%D0%B6%D0%B5%D0%BD%D0%B8%D0%B9-c-null)

|OR|true|false|null|
|---|---|---|---|
|true|true|true|true|
|false|true|false|null|
|null|true|null|null|

|AND|true|false|null|
|---|---|---|---|
|true|true|false|null|
|false|false|false|false|
|null|null|false|null|

|Операция|Результат|
|---|---|
|NOT true|false|
|NOT false|true|
|NOT null|null|

### Неявное приведение типов в PL/Plus

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%92%D1%8B%D1%80%D0%B0%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F-%D0%B8-%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%BE%D1%80%D1%8B#%D0%BD%D0%B5%D1%8F%D0%B2%D0%BD%D0%BE%D0%B5-%D0%BF%D1%80%D0%B8%D0%B2%D0%B5%D0%B4%D0%B5%D0%BD%D0%B8%D0%B5-%D1%82%D0%B8%D0%BF%D0%BE%D0%B2-%D0%B2-plplus)

В первой строке мы видим сравнение двух разных типов — числа и строки. То есть мы перешли к рассмотрению особенности Pl/Plus, дающей возможность применять в выражении значения с разными типами — неявное приведение типов. Важно знать, как это приведение выполняется. В этой строке PL+ приведёт строку к числу и сравнит 2 числа, мы получим результат `true`. Если бы шло сравнение строк, то сравнивались бы первые символы, два больше, чем один и значение флага было бы `false`.

Во второй строке символьное значение 1 приведётся к числовому. В третьей наоборот, числа будут использованы как строки.

Рекомендуется все же избегать неявного приведения. Там, где оно может возникнуть, лучше выполнять явное приведение, обрабатывая ошибки преобразований. Это помогает сделать код надежным и читаемым.

```
flag := '111' > 22; /* true, если бы 22 было бы неявно приведено к строке, то результат был бы false, т.к. '111' < '22' */
tomorow := today + '1'; /* результатом будет завтрашняя дата */
str := 'one: ' || 1 || ', two: ' || 2 || ', three: ' || 3; /* результат будет строкой */
num := '1' + 2; /* получится число '3', так как используется операция +, а не || */
```

### Общие приоритеты операторов

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%92%D1%8B%D1%80%D0%B0%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F-%D0%B8-%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%BE%D1%80%D1%8B#%D0%BE%D0%B1%D1%89%D0%B8%D0%B5-%D0%BF%D1%80%D0%B8%D0%BE%D1%80%D0%B8%D1%82%D0%B5%D1%82%D1%8B-%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%BE%D1%80%D0%BE%D0%B2)

В сложных смешанных выражениях могут присутствовать значения разных типов и операторы различных групп. В зависимости от порядка выполнения операторов может меняться не только результат выражения, но и его тип. Такие выражения вычисляются в соответствии с общим приоритетом операторов. В таблице на вашем мониторе перечислены операторы в порядке убывания приоритета. Наивысший приоритет у скобок, предназначенных именно для изменения последовательности выполнения, совсем как в арифметических выражениях. Выражения, заключенные в скобки выполняются в первую очередь. Далее идут возведение в степень, унарные минус и плюс, умножение и деление, сложение и вычитание, конкатенация, сравнение, логическое отрицание, логическое И, и самый низший приоритет — у логического ИЛИ.

|Оператор|Описание|
|---|---|
|()|скобки позволяют изменить порядок вычисления выражения|
|**|возведение в степень|
|-, +|унарный минус и унарный плюс|
|*, /|умножение и деление|
|+, -|сложение и вычитание|
|две вертикальные черты|конкатенация строк|
|=, !=, <>, <, >, <=, >=, IS null, IS NOT null, LIKE, NOT LIKE, IN, NOT IN|операторы сравнения|
|NOT|логическое отрицание|
|AND|логическое И|
|OR|логическое ИЛИ|

Условные операторы относятся к командам ветвления. Они позволяют выполнять определенную часть кода в зависимости от заданного условия.

### Оператор `if`

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%A3%D1%81%D0%BB%D0%BE%D0%B2%D0%BD%D1%8B%D0%B5-%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%BE%D1%80%D1%8B#%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%BE%D1%80-if)

```
if <условие1> then
    <выполняемая_часть_кода1>
[elsif <условие2> then
    <выполняемая_часть_кода2>]
[elsif <условие3> then
    <выполняемая_часть_кода3>]
...
[else
    <выполняемая_часть_кода>]
end if;
```

Оператор `if` позволяет в зависимости от выполнения условий выполнить ту или иную часть кода. Вот так выглядит его общий синтаксис.

Обязательной является только первая часть и завершающий `end if`.

Остальные добавляются по необходимости.

Условие — это логическая переменная или логическое выражение, результатом которого является `true`, `false` или `null`. Когда все значения в переменной известны, результатом может быть только `true` или `false`. Если же какая-то из переменных не определена (`null`), то и результат может быть не определен. Если необходимо сравнение с `null`, то существует специальные логические выражения `is null` и `is not null`, которые возвращают `true` или `false`. Так же в условиях часто используются функции `nvl` и `coalesce`, которые позволяют вернуть первое непустое значение из двух значений или из множества указанных значений, соответственно.

Как видно из синтаксиса, условий может быть сколь угодно много. Проверка будет происходить от первого к последнему и выполнится только секция кода, соответствующая самому первому условию, вернувшему `true`. Остальные выполнены не будут, даже если результатом логического выражения будет `true`.

Блок с выполняемой частью кода может содержать множество выполняемых команд. В том числе и вложенные условные операторы.

```
if num = 1 then
    --выведем "Один"
    stdio.put_line_buf('Один');
elsif num = 2 then
    --выведем "Два"
    stdio.put_line_buf('Два');
elsif num = 3 then
    --выведем "Три"
    stdio.put_line_buf('Три');
end if;
```

На экране вы видите пример использования оператора `if`.

В PL/Plus реализована логика ускоренного вычисления выражений, которая позволяет не вычислять все условия, если имеющиеся уже позволяют вернуть правильный результат. Например, в `условие1 and условие2` если условие1 вернет `false`, то нет необходимости вычислять второе условие, т.к. общий результат все равно будет `false`. Это может быть важно, если одно из условий требует для вычисления больших затрат — тогда его стоит поставить в конец.

### Оператор `case`

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%A3%D1%81%D0%BB%D0%BE%D0%B2%D0%BD%D1%8B%D0%B5-%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%BE%D1%80%D1%8B#%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%BE%D1%80-case)

Команда `case` по своей логике работы похожа на команду `if`. Она позволяет в зависимости от значения выражения выбрать одну из нескольких возможных выполняемую часть кода.

```
case выражение of
    :<значение1_1>[, <значение 1_2>, ... <значение1_N>]:
        <выполняемая_часть_кода1>
    [:<значение2_1>[, <значение 2_2>, ... <значение2_M>]:
        <выполняемая_часть_кода2>]
    ...
[else
    <выполняемая_часть_кода>]
end;
```

Алгоритм обработки оператора следующий. Сначала происходит вычисление выражения, и далее получившееся значение сравнивается с каждым из значений по порядку. Как только найдется совпадающее значение, выполнится соответствующий блок кода и выполнение команды закончится. Все дальнейшие значения будут проигнорированы. Условие `else`, так же как и у оператора `if`, служит для описания команд, которые необходимо выполнить, если не выполнилось ни одного равенства. В случае отсутствия блока `else` при компиляции автоматически добавится конструкция `else null;`. Блок с выполняемой частью кода может содержать множество выполняемых команд. В том числе и вложенные операторы `case` или `if`.

Предыдущий пример, записанный с помощью оператора case, будет выглядеть вот таким образом:

```
case num of
    :1:
        --выведем "Один"
        stdio.put_line_buf('Один');
    :2:
        --выведем "Два"
        stdio.put_line_buf('Два');
    :3:
        --выведем "Три"
        stdio.put_line_buf('Три');
end;
```

### Выражение `case`

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%A3%D1%81%D0%BB%D0%BE%D0%B2%D0%BD%D1%8B%D0%B5-%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%BE%D1%80%D1%8B#%D0%B2%D1%8B%D1%80%D0%B0%D0%B6%D0%B5%D0%BD%D0%B8%D0%B5-case)

Перейдём к case-выражениям. Они очень похожи на команду `case` и решают ту же задачу, но у них имеется ряд принципиальных различий. Во-первых, case-выражение возвращает конкретное значение определенного типа, а не содержит список исполняемых команд; Во-вторых, результат работы case-выражения всегда записывается в какую-то переменную или используется в запросе.

case-выражения бывают простые и поисковые. Простые похожи на команду `case`: есть выражение, которое сравнивается с рядом значений и при совпадении возвращается соответствующий результат.

Синтаксис простого CASE-выражения:

```
case <выражение>
    when <значение1> then <результат_1>
    [when <значение2> then <результат_2>]
    ...
    [else <результат>]
end
```

Пример использования простого case-выражения:

```
num_string :=
    case num
        when 1 then 'Один'
        when 2 then 'Два'
        when 3 then 'Три'
    end;
```

Поисковые case-выражения отличаются тем, что не содержат одного выражения и множества значений для сравнения. Каждое условие `when` содержит свое собственное логическое выражение или переменную, и при ее равенстве `true` берется соответствующее значение.

Синтаксис поискового case-выражения:

```
case
    when <выражение_1> then <результат_1>
    [when <выражение_2> then <результат_2>]
    ...
    [else <результат>]
end;
```

Пример использования поискового case-выражения:

```
num_string := 'Оценка количества записей: ' ||
    case
        when record_count = 1 then 'одна запись'
        when record_count = 2 then 'две записи'
        when record_count > 2 and record_count <= 10 then 'несколько записей'
        when record_count > 10 then 'много записей'
    end;
```

При отсутствии необязательного блока `else` не происходит его автоматическое создание при компиляции, но оно подразумевается. В этом случае, если не выполняется ни одно из условий `where`, то результат всего case-выражения будет `null`. Таким образом, при отсутствии блока `else` результат такой же, как если бы использовался `else null`.

```
name := name || case when patronym is not null then ' ' || patronym end;
```

Функция — это подпрограмма, которая вычисляет значение по переданным в нее параметрам. Помимо встроенных функций в PL/Plus, как и в любом другом языке программирования, есть возможность создавать пользовательские функции.

В PL/Plus есть несколько видов широко используемых встроенных функций. Это функции для работы с null-выражениями, строковые функции, числовые функции, функции для работы с датами и функции преобразования числа или даты в строку и обратно.

### Функция для работы с null-выражениями

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%92%D1%81%D1%82%D1%80%D0%BE%D0%B5%D0%BD%D0%BD%D1%8B%D0%B5-%D1%84%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D0%B8#%D1%84%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D1%8F-%D0%B4%D0%BB%D1%8F-%D1%80%D0%B0%D0%B1%D0%BE%D1%82%D1%8B-%D1%81-null-%D0%B2%D1%8B%D1%80%D0%B0%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F%D0%BC%D0%B8)

В NVL передается 2 выражения. Если выражение_1 не является NULL, то NVL возвращает его, иначе возвращает выражение_2.

```
NVL(<выражение_1>, <выражение_2>)

name := NVL(client_name, 'n/a');
is_true := NVL(bool_value, true);
```

Количество выражений, передаваемых в COALESCE не ограничено, возвращено будет первое не являющееся NULL.

```
COALESCE(<выражение_1>, <выражение_2>[, <выражение_3>,...<выражение_N>])

cnt := COALESCE(cnt_1, cnt_2, cnt_3, 0);
```

### Строковые функции

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%92%D1%81%D1%82%D1%80%D0%BE%D0%B5%D0%BD%D0%BD%D1%8B%D0%B5-%D1%84%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D0%B8#%D1%81%D1%82%D1%80%D0%BE%D0%BA%D0%BE%D0%B2%D1%8B%D0%B5-%D1%84%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D0%B8)

Функция LENGTH позволяет узнать длину переданной в неё строка_1. Если переданная строка, является NULL значением, то LENGTH вернет NULL.

```
LENGTH(<строка_1>)

result := LENGTH('111');                        -- 3
```

SUBSTR возвращает часть строка_1, начинающуюся с символа с номером a, и имеющую длину b символов. Если b отсутствует, то по умолчанию возвращаются все символы до конца строки начиная с символа с номером a.

В зависимости от знака b направление подсчета количества символов будет отличаться. Если b положительно – подсчет производится слева направо, если отрицательно - то справа налево. Если b равно 0, то возвращается значение NULL.

```
SUBSTR(<строка_1>, <a>[, <b>])

result := SUBSTR('На дворе трава', 4, 5);       -- 'дворе'
```

INSTR возвращает позицию вхождения строка_2 в строка_1. Если указано a, то поиск по строка_1 производится слева направо, начиная с позиции a. Также как и в SUBSTR, Если a отрицательно, то строка_1 просматривается в обратном направлении. При указанном b возвращается позиция указывающая местоположение b-го вхождения. Значением по умолчанию, как для a так и для b является 1, что дает в результате позицию, первого вхождения, строка_2, в строка_1. Если при заданных a и b, строка_2 не найдена, возвращается 0.

```
INSTR(<строка_1>, <строка_2>[, <a>[, <b>]])

result := INSTR('На дворе трава', 'а', 1, 2);   -- 12
```

LPAD возвращает строка_1, дополненную слева до размера х символами строка_2.

RPAD выполняет те же действия справа.

Для обеих функций верны следующие утверждения:

- если размер строка_2, меньше х, то она дублируется, пока конечный результат не достигнет нужного значения;
- если размер строка_2 больше х, то берутся только первые х ее символов;
- если строка_2 не указана, то до указанного размера x строка_1 по умолчанию дополняется пробелами.

```
LPAD(<строка_1>, <х>, <строка_2>)

result := LPAD('На дворе трава', 16, '_');      -- '__На дворе трава'
```

```
RPAD(<строка_1>, <х>, <строка_2>)

result := RPAD('На дворе трава', 16, '_');      -- 'На дворе трава__'
```

LTRIM и RTRIM возвращают строка_1, в которой удалены крайние левые или правые символы соответственно, идентичные символам строка_2. Значением по умолчанию для строка_2 является знак пробела. строка_1 просматривается с левого или правого края, и при встрече первого символа не совпадающего с строка_2, возвращается результат.

```
LTRIM(<строка_1>, <строка_2>)

result := LTRIM('    На дворе трава');          -- 'На дворе трава'
```

```
RTRIM(<строка_1>, <строка_2>)

result := RTRIM('На дворе трава    ');          -- 'На дворе трава'
```

Функция TRIM объединяет в себе LTRIM и RTRIM - удаляет крайние символы одновременно с двух сторон.

```
TRIM(<строка_1>, <строка_2>)

result := TRIM('    На дворе трава    ');       -- 'На дворе трава'
```

UPPER возвращает строка_1, в которой все строчные символы заменены на прописные. LOWER же наоборот заменяет в строка_1 все прописные символы на строчные. Символы не являющиеся буквами не изменяются в обеих функциях.

```
UPPER(<строка_1>)

result := UPPER('На дворе трава');              -- 'НА ДВОРЕ ТРАВА'
```

```
LOWER(<строка_1>)

result := LOWER('НА ДВОРЕ ТРАВА');              -- 'на дворе трава'
```

INITCAP. Возвращает строка_1, в которой каждое слово начинается с прописной буквы и продолжается строчными. Слова разделяются пробелами или не буквенно-цифровыми символами. Символы не являющиеся буквами не изменяются.

```
INITCAP(<строка_1>)

result := UPPER('На дворе трава');              -- 'На Дворе Трава'
```

CONCAT — функция конкатенации строк. Соединяет строка_1 со строка_2 и возвращает результат. То же что и операция "||".

```
CONCAT(<строка_1>, <строка_2>)

result := CONCAT('На дворе', ' трава');         -- 'На дворе трава'
```

REPLACE возвращает строка_1, в которой каждое вхождение строка_поиска, заменяется на строка_замены. Если строка_замены, не указана, то все вхождения строка_поиска, удаляются из строка_.

```
REPLACE(<строка_1>, <строка_поиска>[, <строка_замены>])

result := REPLACE('0000123', '0', '_');         -- '____123'
```

TRANSLATE возвращает строка_1, в которой все вхождения каждого символа строка_поиска замещаются соответствующим символом строка_замены. Функция TRANSLATE, является расширением функции REPLACE. Если строка_поиска длиннее, чем строка_замены, то все ее лишние символы удаляются, поскольку для них нет соответствующих символов в строка_замены. строка_замены не может быть пустой.

```
TRANSLATE(<строка_1>, <строка_поиска>, <строка_замены>)

result := TRANSLATE('1tech23', '123', '456');   -- '4tech56'
```

### Числовые функции

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%92%D1%81%D1%82%D1%80%D0%BE%D0%B5%D0%BD%D0%BD%D1%8B%D0%B5-%D1%84%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D0%B8#%D1%87%D0%B8%D1%81%D0%BB%D0%BE%D0%B2%D1%8B%D0%B5-%D1%84%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D0%B8)

ABS — функция, возвращающая абсолютное значение (значение по модулю) для переданного ей x.

```
ABS(<x>)

result := ABS(-23);             -- 23
```

Функция SIGN. Если x < 0 возвращает -1, x = 0 возвращает 0, x > 0 возвращает 1.

```
SIGN(<x>)

result := SIGN(-23);            -- -1
```

ROUND — функция округления числа. Возвращает х округленное до y разрядов справа от десятичной точки. Значением по умолчанию для y является 0, при этом x округляется до ближайшего целого числа. Если y - отрицательное число, то округляются цифры слева от десятичной точки. y должен быть целым числом.

```
ROUND(<x>[, <y>])

result := ROUND(123.456, 1);    -- 123.5
```

TRUNC возвращает x усеченное (не округленное) до y десятичных разрядов. Значением по умолчанию, для y является 0, при этом x усекается до целого числа. Если y отрицательно, то усекаются цифры слева от десятичной точки.

```
TRUNC(<x>[, <y>])

result := TRUNC(123.456, 1);    -- 123.4
```

CEIL возвращает наименьшее целое число, большее или равное х.

```
CEIL(<x>)

result := CEIL(32.65);          -- 33
```

FLOOR возвращает наибольшее целое число, меньшее или равное х.

```
FLOOR(<x>)

result := FLOOR(32.65);         -- 32
```

MOD возвращает остаток от деления на . Может быть использована для отсчета, например, каждой сотни экземпляров с целью вывода информационного сообщения.

```
MOD(<x>, <y>)

result := MOD(365, 100);         -- 65
```

Помимо упомянутых числовых функций в PL+ поддерживаются и некоторые другие, но на практике они не используются или используются крайне редко. Это такие функции как SQRT, POWER, LOG, LN, EXP, COS, SIN и другие.

### Функции для работы с датами

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%92%D1%81%D1%82%D1%80%D0%BE%D0%B5%D0%BD%D0%BD%D1%8B%D0%B5-%D1%84%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D0%B8#%D1%84%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D0%B8-%D0%B4%D0%BB%D1%8F-%D1%80%D0%B0%D0%B1%D0%BE%D1%82%D1%8B-%D1%81-%D0%B4%D0%B0%D1%82%D0%B0%D0%BC%D0%B8)

Рассмотрим функции для работы с датами. Некоторые из них похожи на числовые функции и несут то же смысловое значение, но при использовании с датами имеют немного иной набор параметров.

SYSDATE. Возвращает текущую дату и время в системе. Возвращаемый тип данных — DATE

```
SYSDATE

result := SYSDATE;                                  -- 21/07/2020 18:52:02
```

TRUNC Возвращает дату d усеченную до указанного формата.

```
TRUNC(<d>[, <формат>])

result := TRUNC(TO_DATE('31/07/2020'), 'YEAR');     -- 01/01/2020
```

ROUND возвращает дату d округленную до указанного формата.

```
ROUND(<d>[, <формат>])

result := ROUND(TO_DATE('31/07/2020'), 'YEAR');     -- 01/01/2021
```

Формат для функций TRUNC и ROUND соответствует следующей таблице:

|Формат|Единица округления или усечения|
|---|---|
|СС, SCC|Век|
|SYYYY, YYYY, YEAR, SYEAR, YYY, YY, Y|Год (округляется до 1 июля)|
|IYYY, IYY, IY, I|Год ISO|
|Q|Квартал (округляется до шестнадцатого дня второго месяца квартала)|
|MONTH, MON, MM, RM|Месяц (округляется до шестнадцатого дня)|
|WW|То же день недели, что и первый день года|
|IW|То же день недели, что и первый день года ISO|
|W|То же день недели, что и первый день месяца|
|DDD, DD, J|День|
|Day, DY, D|Первый день недели|
|HH, HH12, HH24|Час|
|MI|Минута|

Если формат не указан, применяется формат по умолчанию 'DD', который округляет d до ближайшего дня.

ADD_MONTHS возвращает дату d плюс x месяцев. Значение x может быть любым целым числом. Если в месяце, полученном в результате, число дней меньше, чем в месяце d, то возвращается последний день месяца результата. Если число дней не меньше то день месяца результата и день месяца d совпадают. Временные компоненты даты d и результата одинаковы.

```
ADD_MONTHS(<d>, <x>)

result := ADD_MONTHS('01/08/2020', 3);              -- 01/11/2020
```

LAST_DAY возвращает дату последнего дня того месяца, в который входит d. Эту функцию, можно применять для определения количества дней оставшихся в текущем месяце.

```
LAST_DAY(<d>)

result := LAST_DAY('25/07/2020');                   -- 31/07/2020
```

MONTHS_BETWEEN возвращает число месяцев между дата_1 и дата_2. Если дни в дата_1 и дата_2 совпадают или если обе даты являются последними днями своих месяцев, то результат представляет собой целое число. В противном случае результат будет содержать дробную часть, по отношению, к 31-дневному месяцу.

```
MONTHS_BETWEEN(<дата_1>, <дата_2>)

result := MONTHS_BETWEEN('02/08/2020', '02/06/2020'); -- 2
```

Арифметические действия с датами:

Разница в днях между d1 и d2. Это значение является действительным числом, где дробная часть означает неполный день.

```
d1 - d2

result := to_date('02/08/2020','dd/mm/yyyy') - to_date('02/06/2020','dd/mm/yyyy');              -- 61
```

Прибавление дней. К d1 добавляется n дней и возвращается результат, имеющий тип DATE. n может быть вещественным числом содержащим не полный день.

```
d1 + n

result := TO_DATE('02/08/2020', 'dd/mm/yyyy') + 60; -- 01/10/2020
```

Вычитание дней. Из d1 вычитается n дней и возвращается результат, имеющий тип DATE. n может быть вещественным числом содержащим не полный день.

```
d1 - n

result := TO_DATE('02/08/2020', 'dd/mm/yyyy') - 60; -- 03/06/2020
```

### Функции преобразования

[](https://github.com/kuzminem/cft-plplus/wiki/%D0%92%D1%81%D1%82%D1%80%D0%BE%D0%B5%D0%BD%D0%BD%D1%8B%D0%B5-%D1%84%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D0%B8#%D1%84%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D0%B8-%D0%BF%D1%80%D0%B5%D0%BE%D0%B1%D1%80%D0%B0%D0%B7%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F)

Эти функции используются для преобразования чисел и дат в строки и наоборот.

Функции преобразования числа в строку и обратно:

```
TO_CHAR(<число_1>[, <формат>[, <nls_параметр>]])
```

Функция TO_CHAR преобразует число_1 в строку в соответствии с заданным форматом. nls_параметр управляет выбором языка, но, как правило, достаточно языка по умолчанию.

```
TO_NUMBER(<строка_символов>[, <формат> [, <nls_параметр>]])
```

Функция TO_NUMBER делает обратное TO_CHAR преобразование - из строки в число в соответствии с заданным форматом.

Описание форматов:

|Элемент формата|Пример строки данного формата|Описание|
|---|---|---|
|9|99|Каждая цифра 9 представляет значащую цифру результата. Число значащих цифр возвращаемого значения равно числу цифр 9, отрицательное значение предваряется знаком минуса. Все начальные нули заменяются пробелами|
|0|0999|Возвращается число с начальными нулями, а не пробелами|
|0|9990|Возвращается число с конечными нулями, а не пробелами|
|$|$999|Возвращаемое значение предваряется знаком доллара не зависимо от используемого символа денежной единицы можно применить совместно с начальными или конечными нулями|
|B|B999|Вместо нулевой целой части десятичного число возвращаются пробелы|
|MI|999MI|Возвращает отрицательное число, у которого знак минуса указан не в начале, а в конце. В положительном значении на этом месте будет пробел|
|S|S9999|Возвращаемое число предваряется знаком: + для положительных, чисел - для отрицательных|
|S|9999S|Возвращаемое число заканчивается знаком: + для положительных, чисел - для отрицательных|
|PR|99PR|Возвращается отрицательное число в угловых скобках "<", ">". У положительных чисел, на этом месте пробелы|
|D|99D9|Возвращает число с десятичной точкой в указанной позиции. Число 9 с обеих сторон указывает максимальное число цифр|
|G|9G999|Возвращает число с разделителем групп в указанной позиции. G может появляться в указанной строке формата неоднократно|
|C|C99|Возвращает число с символом денежной единицы ISO в указанной позиции. C может появляться в указанной строке формата неоднократно|
|L|L999|Возвращает число с символом денежной единицы национального языка в указанной позиции|
|,|999,999|Возвращает число с запятой в указанной позиции, не зависимо от выбранного разделителя групп|
|.|99.99|Возвращает число с десятичной точкой в указанной позиции, не зависимо от выбранного десятичного разделителя|
|V|99V999|Возвращает число, умноженное на 10n, где n - это число цифр 9 после V. При необходимости значение округляется|
|EEEE|9.99EEEE|Возвращает число в экспоненциальном представлении|
|RM|RM|Возвращает число при помощи римских цифр верхнего регистра|

Примеры:

```
result := to_char(1234.56, '9999.9');               -- '1234.6'

result := to_char(1234.56, '9,999.99');             -- '1,234.56'

result := to_char(12, '000099');                    -- '000012'

result := to_number('1234.6', '9999.9');            -- 1234.6

result := to_number('1,234.56', '9,999.99');        -- 1234.56

result := to_number('000012', '000099');            -- 12
```

Функции преобразования даты в строку и обратно:

```
TO_CHAR(<дата>, [, <формат>[, <nls_параметр>]])
```

Функция TO_CHAR, в которую передана дата, преобразует дату в строку в соответствии с заданным форматом.

```
TO_DATE(<строка_символов>[, <формат>[, <nls_параметр>]])
```

Функция TO_DATE делает обратное TO_CHAR преобразование — из строки в дату в соответствии с заданным форматом.

Описание форматов:

|Элемент формата|Описание|
|---|---|
|AD, A.D.|Показатель "нашей эры" (с точками или без точек)|
|AM, A.M.|Показатель времени до полудня (с точками или без точек)|
|BC, B.C.|Показатель "до нашей эры" (с точками или без точек)|
|CC, SCC|Век SCC возвращает даты "до нашей эры" как отрицательные значения|
|D|День недели (1-7)|
|DAY|Название дня, дополненное пробелами до девяти символов *|
|DD|День месяца (1 - 31)|
|DDD|День года (1 - 366)|
|DY|Сокращенное название дня *|
|IW|Неделя года (в основе лежит стандарт ISO)|
|IYY, IY, I|Последние три, две или одна цифра года ISO|
|IYYY|Четырех цифровое обозначение года, основанное на стандарте ISO|
|HH, HH12|Час дня (1-12)|
|HH24|Час дня (0-23)|
|J|День по Юлианскому календарю. Число дней с 1 января 4712 года до н.э. Соответствующий результат будет целым значением|
|MI|Минута (0-59)|
|MM|Месяц (1 - 12)|
|MONTH|Название месяца, дополненное пробелами до девяти символов *|
|MON|Сокращенное название месяца *|
|PM, P.M.|Показатель времени после полудня (с точками или без точек)|
|Q|Квартал года (1 - 4)|
|RM|Месяц, обозначенный римскими цифрами (I - XII)|
|RR|Последние две цифры года для других веков|
|SS|Секунды (0-59)|
|SSSS|Секунды после полуночи (0 - 86399)|
|WW|Неделя года (1-53). Неделя 1 начинается с первого дня года и продолжается до седьмого дня. таким образом, недели не всегда начинаются с воскресенья|
|W|Неделя месяца (1-5)|
|Y, YYY|Год с запятой в указанной позиции|
|YEAR, SYEAR|Год, записанный буквами, SYEAR возвращает даты до нашей эры как отрицательные значения *|
|YYYY, SYYYY|Четырех цифровой год. Возвращает даты до нашей эры как отрицательные значения|
|YYY, YY, Y|Последние три, две или одна цифра года|
|Знак пунктуации|Все знаки пунктуации дублируются в результирующей строке символов|
|"Текст"|Текст, заключенный в двойные кавычки так же дублируется|

Некоторые форматы при преобразовании дат чувствительны к регистру (они в таблице имеют значок *). Например, функция MON вернет JAN, а если записать Mon, то получите Jan

Примеры:

```
result := TO_CHAR(sysdate, 'yyyy/mm/dd HH24:MI:SS');                -- '2020/08/04  20:04:52'

result := TO_CHAR(sysdate, 'DD Mon YYYY');                          -- '05 Авг 2020'

result := TO_CHAR(sysdate, 'FMDD Mon YYYY');                        -- '5 Авг 2020'

result := TO_DATE('2020/08/04  20:04:52', 'yyyy/mm/dd HH24:MI:SS'); -- 04/08/2020 20:04:52

result := TO_DATE('05 Авг 2020', 'DD Mon YYYY');                    -- 05/08/2020

result := TO_DATE('5 Авг 2020', 'FMDD Mon YYYY');                   -- 05/08/2020
```

Функции и процедуры — это подпрограммы, которые выполняют какие-то определённые действия.

Функции и процедуры играют важную роль в реализации модульного подхода. При реализации бизнес-процесса или небольшой задачи рекомендуется код разбивать на функции и процедуры. Вместо того, чтобы писать один и тот же запрос снова и снова, поместите его в функцию и вызовите эту функцию в нужных местах. Такой код создает меньше проблем с отладкой, оптимизацией, сопровождением и развитием. Также при большом объёме кодирования можно разбивать программу на логические куски, если они слабо связаны, и выделять их в процедуры и функции. Это увеличит прозрачность кода.

Одно из отличий функции от процедуры заключается в том, что функция возвращает результат, а процедура — нет. Таким образом, процедуры относятся к подпрограммам, обеспечивающим оптимизацию и повторное использование программной логики. Функция, в отличии от процедуры, не является самостоятельным оператором, поэтому вызов функции всегда является частью исполняемого оператора, то есть включается в выражение или служит в качестве значения по умолчанию, присваиваемого параметру или переменной при объявлении.

Данные хранятся в базе данных.

Все ТБП, массивы и структуры являются таблицами базы данных, а их реквизиты — полями.

Для обращения к данным БД используется модификация языка структурированных запросов SQL, называемая оператором выборки или PL/PLUS-выборкой.

Синтаксис PL/PLUS-выборки будет понятен любому человеку, знакомому с SQL, но есть небольшие отличия, которые мы рассмотрим далее:

```
select [distinct] < переменная-алиас >
( < select-лист > ) in < источник данных >
[where < логическое выражение – условие выборки >]
[connect [nocycle] by < условие иерархического соединения >
   [start < условие выборки 0-уровня иерархии >] ]
[group by < выражение группировки 1 > [,< выражение группировки 2 > ...] ]
   [having < логическое выражение – групповое условие >] ]
[order by < выражение сортировки 1 > [, < выражение сортировки 2>... ] ]
```

Обязательными являются только секции, которые не заключены в квадратные скобки, т. е. `select` и `in`. Они соответствуют секциям `select` и `from` SQL-запросов, но имеют некоторые отличия в построении.

Секция `select` состоит из select-листа с переменной-алиасом. Select-лист содержит выбираемые поля через запятую. Выражения выборок могут являться реквизитами источника данных, константами, обращениями к переменным и параметрам операции (как локальным, так и глобальным), функциями. Используемые функции при этом не должны модифицировать данные в БД. Выражение выборки тоже может являться выборкой, но только скалярной, то есть возвращающий не более одной записи и содержащей только одно выбираемое значение.

Переменная-алиас используется для выражений выборки, являющимися реквизитами источников данных из секции `in`. Также используется для определения условий, накладываемых на источник в секции `where`. Алиас без указания выражения выборки является ссылкой на экземпляр источника (строку таблицы).

```
[< переменная-алиас 1 >.]< выражение выборки 1 > [: < алиас 1 > ]
   [, [< переменная-алиас N >.]< выражение выборки 2 > : < алиас 2 > ...]
```

Секция `in` содержит список источников данных:

```
< источник 1 > [, (< источник 2 > [{all, collections}] : < переменная-алиас >...] [{all, collections}]
```

Источником обычно является ТБП или реквизит-коллекция. Переменной-алиасом источника 1 является указанная перед селект-листом переменная. Для остальных источников алиасы указываются через разделитель-двоеточие после имени соответствующего источника. В качестве источников данных также могут использоваться выборки и заранее описанные статические курсоры. Если в качестве источников используются ТБП, участвующие в коллекциях, то могут использоваться опции `all` и `collections` указывающие на необходимость поиска по всем экземплярам типа или только по коллекциям, соответственно. Если опции не заданы, то поиск производится только по не входящим ни в какие коллекции экземплярам. Для первого источника эти опции указываются в конце всего списка, для остальных — непосредственно после объявления источника.

Также ключевой секцией является `where`, в которой могут быть заданы дополнительные условия выборки с помощью операторов `and` и `or`.

Дополнительные опции для работы с выборками PL/Plus:

- `distinct` — оставляет в результирующей выборке только уникальные строки.
- `connect [nocycle] by … start` — секция для построения иерархических запросов. В случае использования — в `in` может быть указан только один тип или коллекция.
- `group by … [having]` — позволяет группировать выборку по значению одного или нескольких выражений. `having` задаёт дополнительные условия группировки.
- `order by` — сортирует выборку по одному или нескольким выражениям. Приоритет сортировки определяется порядком объявления выражений. Поддерживает опции `asc` и `desc`, задающие направление по возрастанию и убыванию соответственно, а также `nulls first` и `nulls last`, которые переносят пустые значения в начало или в конец.

Давайте рассмотрим пример: выборка ID, кода филиалов и их наименований, для филиалов открытых после 1 января 2020 года включительно:

```
select d
( d.ID      : C_ID
, d.CODE    : C_CODE
, d.NAME    : C_NAME
)
in ::[BRANCH]
where d.[DATE_OPENNING] >= to_date('01/01/2020', 'dd/mm/yyyy')
```

- алиасом таблицы здесь является "d", он указывает на ТБП BRANCH;
- алиас таблицы указывается при перечислении полей, при выборке из нескольких таблиц, указывается алиас соответствующей таблицы;
- два двоеточия перед ключевым словом BRANCH является маркером, того, что это ТБП или справочник, (структуры и массивы просто заключаются в квадратные скобки);
- в условии, накладываемом на поля таблицы, обращение к этим полям тоже происходит через алиас таблицы.

В качестве логических выражений условий выборки (where) можно использовать также следующие операторы, использующие подзапросы:

- оператор существования выборки:

```
exists (<PL/Plus-выборка>)
```

- расширенные операторы сравнения-включения:

```
<выражение>  > {any, all} (<PL/Plus-выборка>)
<выражение>  >= {any, all} (<PL/Plus-выборка>)
<выражение>  < {any, all} (<PL/Plus-выборка>)
<выражение>  <= {any, all} (<PL/Plus-выборка>)
(<выражение 1>, <выражение 2>, …) [not]  in (<PL/Plus-выборка>)
(<выражение 1>, <выражение 2>, …)  = {any, all} (<PL/Plus-выборка>)
(<выражение 1>, <выражение 2>, …)  != {any, all} (<PL/Plus-выборка>)
(<выражение 1>, <выражение 2>, …)  <> {any, all} (<PL/Plus-выборка>)
```

По умолчанию используется внутреннее соединение источников данных (inner join). Будут выбраны строки таблицы, для которых оператор сравнения вернул Истину. Для использования внешнего соединения (outer join) применяется оператор (true). В этом случае к результату предыдущего варианта будут добавлены не попавшие в inner join строки таблицы, к реквизиту которой не был применён оператор (true). В примере ниже это источник с переменной-алиасом base_table. Внешнее соединение:

```
<выражение 1 < переменная-алиас 1 >.< реквизит 1 >> <оператор сравнения>
   <выражение 2 < переменная-алиас 2 >.< реквизит 2 >>(true)
```

Пример внешнего соединения:

```
where base_table.[FIELD] = add_table.[POSSIBLE_FIELD](true)
```

В PL/Plus выборке могут использоваться также следующие SQL-выражения:

- case-выражения (SQL searched case), позволяющие использовать логическое ветвление (аналогично условному оператору) в секциях when:

```
case
when <условие 1> then <результирующее выражение 1>
[when <условие 2> then <результирующее выражение 2>]
[...]
[else <результирующее выражение>]
end
```

- простые case–выражения (SQL simple case), сравнивающие одно выражение с заданными вариантами в секциях when:

```
case <выражение>
when <значение выражения 1> then <результирующее выражение 1>
[when <значение выражения 2> then <результирующее выражение 2>]
[...]
[else <результирующее выражение>]
end
```

PL/Plus выборка может быть построена из нескольких полноценных выборок или коллекций, как результат их объединения. Для этого используются следующие операторы:

- union — объединяет все строки выборок за исключением идентичных.
- union all — объединяет все строки, не исключая идентичные.
- При объединении выборок типы и порядок выбираемых выражений должны совпадать.
- intersect — возвращает только идентичные сроки двух выборок.
- minus — исключает из первой выборки строки, идентичные строкам второй. PL/Plus-выборки используются в циклах или курсорах, о которых будет рассказано позже, сейчас же рассмотрим вариант записи результата выборки в переменную с помощью оператора SELECT INTO:

```
<PL/Plus-выборка>
into <переменная назначения 1>[, <переменная назначения 2> ...];
```

Рассмотрим пример:

```
select d
( d         : C_ID
, d.[CODE]  : C_CODE
, d.[NAME]  : C_NAME
)
in ::[BRANCH]
where d.[DATE_OPENNING] >= to_date('01/01/2020', 'dd/mm/yyyy')
fetch 1
into rBranchID, vBranchCode, vBranchName;
```

В примере происходит выборка трех полей, соответственно, и после INTO должны быть указаны три переменные: пусть это будет rBranchID, vBranchCode и vBranchName, обратите внимание, что переменные в которые считываются значения полей должны совпадать по типу с полями, иначе могут возникнуть ошибки.

Условие с `fetch` использовано для ограничения выборки единственной строкой, поскольку попытка записи в скалярные переменные вызовет ошибку too_many_rows. Обратите внимание, если вы попытаетесь использовать сортировку для выбора нужной строки, она не повлияет на результат. Поскольку сначала накладываются условия выборки и, в частности, условие на количество строк. И только потом выполняется сортировка. Для получения корректного результата необходимо выполнить сортировку в подзапросе, и уже из него выбрать первую строку.

Примечание! БД Oracle может использовать для ограничения количества строк условие `rownum<2`

Мы рассмотрели оператор PL/SQL-выборки, позволяющий получить данные из БД. Но как их туда сохранять, изменять уже сохраненные и удалять?

Для этого существуют несколько операторов изменения данных.

Для записи новых данных в БД используется оператор вставки. Он бывает трёх типов — простой и SQL с прямым указанием значений или с использованием подзапроса.

Рассмотрим простой оператор вставки.

```
insert [ (< дополнительные параметры >) ] < переменная > into <{тип, коллекция}>;
```

Переменная может являться структурой, массивом структур, ссылкой, либо массивом ссылок. В случае структур происходит вставка записей в указанный тип, либо коллекцию. В случае ссылок - перенос существующих записей, например из типа в указанную коллекцию. Пример простого оператора вставки:

```
insert recHistoryState into p_docRef.[HISTORY_STATE];
```

Структура recHistoryState добавится в коллекцию HISTORY_STATE, принадлежащую ссылке p_docRef. Т.е. в ТБП ::[HISTORY_STATES] появится запись с реквизитами recHistoryState, у которой реквизит collection_id будет равен реквизиту [HISTORY_STATE] строки таблицы ::[MAIN_DOCUM], на которую указывает ссылка p_docRef.

Далее посмотрим на синтаксис SQL-оператора вставки прямым указанием значений:

```
insert into <{тип, коллекция}>
[< внутренний цикл >] < переменная-алиас >(
   < выражение назначения 1 > = < выражение 1 >
[, < выражение назначения 2 > = < выражение 2 >...]
) [where < логическое выражение >]
[return < выражение выборки 1 > [, < выражение выборки 2 >...]
   into < переменная-приемник 1 > [,< переменная-приемник 2 >...] ] ;
```

< Переменная-алиас > служит ссылкой на ТБП для описания выражений в операторе. Если в операторе указано условие `where`, тогда при вставке будет происходить проверка на соответствие этому условию вставляемых значений. С помощью ключевых слов `return ... into` можно получить значения выражений после вставки, которые помещаются в переменные-приемники. Если переменными-приемниками являются массивы, тогда они заполняются сразу для всех созданных записей, если же приемники — скалярные переменные, тогда оператор вставки может создать не более одной записи. Пример:

```
insert into ::[USR_GROUPS]
   Rec(Rec.[FGROUP] = V_ADD_FG_REF)
   return Rec%id into V_GROUPS%id;
```

Синтаксис SQL-оператора вставки через PL/Plus подзапрос отличается наличием подзапроса вместо пар < выражение назначения 1 > = < выражение 1 >.

```
insert into <{тип, коллекция}>
[< внутренний цикл >] < переменная-алиас >(
(< выражение назначения 1 > [,< выражение назначения 2 >...] )
= (< PL/Plus подзапрос >)
) [where < логическое выражение >]
[return < выражение выборки 1 > [, < выражение выборки 2 >...]
   into < переменная-приемник 1 > [,< переменная-приемник 2 >...] ] ;
```

Приведём пример:

```
insert into ::[DEPART_PLUS]
    dp((dp%collection, dp.[COUNTER])=
    (select n(n.[COUNTERS], cntr) in ::[NAME_PAYDOC]));
```

Здесь значение переменной cntr и реквизиты строк, выбранных из ::[NAME_PAYDOC], будут указаны в создаваемых записях ТБП ::[DEPART_PLUS].

Чтобы изменить данные, которые уже хранятся в БД используется оператор изменения:

```
update [<внутренний цикл>] <переменная-алиас>(
<выражение назначения 1> = <выражение 1>
[, <выражение назначения 2> = <выражение 2> … ]
[, (<выражение назначения 3> [, <выражение назначения 4> ... ] )
= (<PL/Plus-выборка>) … ]
) in <{тип коллекция}>[({<числовая константа> ’ALL’})][{all collections}[nostatic]]
[where <логическое выражение>]
[return <выражение выборки 1>[,< выражение выборки 2>…]
into <переменная-приемник 1>[,<переменная-приемник 2>…]
```

Оператор изменения отчасти похож на выборку &mdash здесь также есть переменная-алиас, список с использованием этой переменной, секция `in` и условие `where`.

Список, следующий за переменной-алиасом служит не для перечисления выражений выборки, а для указания инструкций модификации. По сути это список присвоений, где в каждом присваивании слева реквизит ТБП, указанный с помощью переменной-алиаса.

Секция `in` указывает какие ТБП мы хотим изменить. В отличие от выборки в секции `in` указывается 1 ТБП.

Часто изменениям подвергаются не все записи таблицы, а только небольшая их часть. Для ограничения изменяемых записей используется условие указываемое в `where`. Изменениям подвергаются только те записи, для которых условие выполняется, как и в `select`. Если `where` не используется, то изменения применятся ко всем записям целевой таблицы.

При использовании выражений в списке присваивания и условий, непредсказуемо определяющих набор изменяемых записей, бывает необходимо получить значения, установленные записям в БД оператором `update`. Для этого служат секции `return` и `into`. Как в `select`, список выражений в `return` определяет, что будет возвращено, а секция `into` определяет куда. В `into` указываются переменные-массивы, т.к. `update` изменяет сразу множество записей, для каждой из которых вычисляются выражения `return`.

Пример:

```
update p(p.[CODE] = upper(p.[CODE]) in ::[BRANCH]
   where p.[CODE] != upper(p.[CODE];
```

Здесь реквизит Код (CODE) экземпляров уже знакомого нам ТБП BRANCH приводится к верхнему регистру (upper), но только в тех записях, где он еще не приведен (p.[CODE] != upper(p.[CODE]))

Запрос удаления данных выглядит проще:

```
delete [<внутренний цикл>] <переменная-алиас>
in <{тип коллекция}>[({<числовая константа> ’ALL’})] [{all, collections} [nostatic]]
[where <логическое выражение>]
[return <выражение выборки 1>[, < выражение выборки 2> ...]
into <переменная-приемник 1>[, <переменная-приемник 2> ...] ]
;
```

В нем уже нет списка присваиваний, потому что записи удаляются целиком. Как и в update, удаление применяется только к записям, для которых выполнено условие, указанное в where. Если условие не выполнится ни для одной записи, то ничего не удалится. Если не указывать секцию where, то удалятся все записи.

Как и в случае с update, при удалении можно получить информацию, что именно мы удалили и опять-таки для этого есть return и into. Использование аналогично update.

Теперь приведем пример удаления, опять же с нашими филиалами, например, удалим все филиалы, код которых является пустым:

```
delete br in ::[BRANCH] where br.[CODE] is null;
```

для того, чтобы изменения сохранились в базе, не забываем использовать команду commit. В PL+ она автоматически выполняется после завершения головной операции.
