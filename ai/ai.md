MCP - для некоторых задач, важно, чтобы норм работало подключать нужную MCP 

MCP (Model Context Protocol) — это открытый стандарт от компании Anthropic, который работает как универсальный мост между ИИ-моделями (агентами) и внешними источниками данных. Простыми словами, это «USB-разъем для ИИ», который позволяет языковым моделям подключаться к файлам, базам данных и API без написания уникальных интеграций для каждого сервиса.Зачем нужен MCP?До появления MCP языковые модели работали изолированно: они знали только то, на чем были обучены. Чтобы дать ИИ доступ к вашей CRM-системе, локальным документам или кодовой базе, разработчикам приходилось писать кастомные скрипты для каждого случая.MCP решает эту проблему, предлагая единый стандарт, состоящий из трех главных компонентов:Инструменты (Tools): Позволяют агенту совершать действия (например, создать задачу в Jira, выполнить поиск по репозиторию, отправить письмо).Ресурсы (Resources): Дают ИИ доступ к данным и документам (базам знаний, логам) для анализа прямо в процессе работы.Промпты (Prompts): Готовые шаблоны запросов, которые разработчики или пользователи задают для ускорения типовых задач.Как это работает на практикеЛокальный доступ к коду (Coding): Популярные ИИ-редакторы (например, Cursor или Windsurf) используют MCP-серверы для прямого чтения вашего кода, папок проекта и выполнения команд в терминале.Работа с документами: Агент может подключиться к Notion или вашим локальным файлам в формате PDF, Markdown и проанализировать их, выдав точный ответ.Взаимодействие с корпоративными системами: ИИ-агенты могут подключаться к базам данных (PostgreSQL и др.) и API бизнес-инструментов, чтобы агрегировать данные прямо из отчетов.Основные преимуществаУниверсальность: Один раз настроив MCP-сервер, его можно использовать в разных ИИ-приложениях (например, в Claude Desktop, ChatGPT).Безопасность: Данные не уходят на обучение самой модели. Агент обращается к серверу в реальном времени и оперирует только тем контекстом, который вы ему разрешили видеть.Многоагентные системы (Multi-agent): MCP стандартизирует обмен данными не только между ИИ и внешним миром, но и между разными ИИ-агентами.

Skills (навыки) у AI-агентов — это переносимые, специализированные пакеты инструкций, сценариев и контекста, которые обучают нейросеть конкретным узким задачам. Они превращают модель общего назначения в эксперта с определенной ролью, выполняя функцию регламентов или шаблонов.Как работают AI-скиллыОбычно скилл представляет собой отдельный структурированный файл (чаще всего SKILL.md) или директорию, которые загружаются в память агента по мере необходимости.Основные элементы скилла включают:Ролевую инструкцию: четко описывает, кто агент, в какой манере он общается и какую роль выполняет.Пошаговый регламент: алгоритм действий (что делать при запросе, какие этапы соблюдать).Примеры (Few-shot): примеры эталонного выполнения задачи, плохие варианты (чего избегать).Интеграции и инструменты: API, функции, или плагины, к которым агент получает доступ.Почему их используют вместо больших промтовЭкономия контекстного окна: Модели не нужно держать в памяти огромный промт со всеми инструкциями. Скиллы активируются динамически — агент «подгружает» нужный навык только тогда, когда берется за соответствующую задачу.Переносимость и версионность: Навык можно использовать повторно, переносить от одного агента к другому, передавать коллегам или хранить в репозитории.Фокус на экспертизе: Вместо универсального ассистента вы создаете набор четких профессиональных ролей, где каждый скилл отвечает за отдельный бизнес-процесс (например, SEO-оптимизация, написание кода или проверка документов).
